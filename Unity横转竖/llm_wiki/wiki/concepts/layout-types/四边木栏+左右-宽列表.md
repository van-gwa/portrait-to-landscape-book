# 四边木栏+左右-宽列表

## 定义

**四边木栏+左右-宽列表**是指UI周围有完整的四边装饰框（木栏包边），左侧为导航标签页（Tab/Toggle），右侧为宽度充足的商品/物品列表的布局方式。适用于商店、选择界面等需要分类展示大量可滚动内容的场景。

## 布局结构

```
┌─────────────────────────────────────┐
│  上边框                              │
├──────┬────────────────────────────┤
│ 左   │  右侧宽列表                 │
│ 标   │  ┌──────┬──────┬──────┐    │
│ 签   │  │Item 1│Item 2│Item 3│ ↕  │
│ 页   │  ├──────┼──────┼──────┤ 可  │
│      │  │Item 4│Item 5│Item 6│ 滚  │
│ 分   │  └──────┴──────┴──────┘ 动  │
│ 类   │                          │
├──────┼────────────────────────────┤
│  下边框                            │
└─────────────────────────────────────┘
```

## 主要特征

| 特征 | 说明 |
|------|------|
| **四边木栏** | 上下左右都有装饰性边框（通常为木栏纹理） |
| **左侧导航** | 标签页/Toggle 组件（TglGroupFun），用于内容分类切换 |
| **右侧宽列表** | 商品/物品列表，宽度≥1000（占背景宽的80%+） |
| **可滚动** | 列表内容超过容器高度时支持纵向滚动 |
| **无形象展示** | 不包含立绘、角色头像等大型形象区域 |
| **无标题区** | 没有独立的上方标题块 |

## 标准检测方法

### 1. 检查木栏包边（必要条件）
```
✓ 节点里有 4 个边框 Image（上下左右）
✓ 都引用 atlas_common_static.png 的边框资源
✓ 四边都能命中（top=✔ bottom=✔ left=✔ right=✔）
```

### 2. 识别左侧标签页（必要条件）
```
✓ 左区域有 TglGroupFun 或 Toggle Group 组件
✓ 节点名包含 "Tgl" 或 "Tab" 关键字
✓ 有多个子 Toggle（商品分类选项）
```

### 3. 识别右侧宽列表（必要条件）
```
✓ 右中区有一个 ScrollRect 或 List 容器
✓ 容器宽度 ≥ 1000（w ≥ background_width × 0.8）
✓ Content 内有多个同类的物品卡片子节点
```

### 4. 排除其他类型（必要条件）
```
✗ 无立绘/形象展示区（不是"左右-宽图"）
✗ 无独立标题块（不是"标题+左右"）
✗ 列表是纯文本且单列（不是"左标签页+右文本列表"）
```

## 横竖屏适配要点

### 竖屏（Portrait）→ 横屏（Landscape）

| 阶段 | 处理方式 | 说明 |
|------|--------|------|
| **左侧标签页** | 改为上方标签栏或下方抽屉菜单 | 减少水平空间占用，改为横向或隐藏式 |
| **右侧列表重排** | 保持纵向滚动，增加列宽或列数 | 横屏宽度充足，可显示更多列 |
| **木栏包边调整** | 上下边框缩小，左右边框扩大 | 适应横屏的长宽比变化 |
| **整体高度** | 缩小总高，尽量在视口内完整展示 | 横屏高度受限，需精简竖向空间 |

### 具体改版方案

#### 方案A：标签页改为上方标签栏（推荐）
```
竖屏：                  横屏：
┌──────────┐           ┌──────────────────┐
│ 上边框   │           │ 上边框 + 标签栏   │
├┐       ┌┤           ├──────────────────┤
││ 列表 │││           │  宽列表         │
││      │││  →         │  支持多列或     │
││      │││           │  更宽的展示     │
├┘       └┤           ├──────────────────┤
│ 下边框   │           │ 下边框          │
└──────────┘           └──────────────────┘
```

#### 方案B：标签页改为左侧抽屉
```
平时：                  打开时：
┌─────────┐           ┌─────────┐
│ 列表    │  →        │▼|列表   │
│         │           │ │       │
└─────────┘           └─────────┘
  点击菜单            左侧滑出标签页
```

#### 方案C：标签页改为下方底部菜单
```
竖屏：                  横屏：
┌──────────┐           ┌──────────────────┐
│ 列表     │           │  宽列表          │
├┐       ┌┤     →      │                 │
││标签页 │││           └──────────────────┘
│└───────┘│           [下方紧凑标签栏]
└──────────┘           
```
