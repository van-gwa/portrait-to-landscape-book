# 左标签+右竖列表

## 定义

「左标签+右竖列表」是左标签系列的基础类型——**左侧竖向标签切换 + 右侧竖列表**，**无顶部贯穿全宽的标题栏**。共14个UI。

**与「T型-上标题-左标签-右竖列表」的核心区别**：本类型**没有贯穿全宽的T型顶部标题栏**。

## 布局结构

```
┌──┬──────────────────┐
│  │  [Item 1]     │
│标│  [Item 2]     │  ← 竖列表（上下滚动）
│签│  [Item 3]     │
│  │  ...          │
└──┴──────────────────┘
   无T型顶部标题！
```

## 三步判断法（5样本+2验证，盲测2/2通过）

### 第1步：左侧有没有竖向标签？

- 左侧有竖向排列的标签/按钮（用于切换分类）→ 继续
- 无 → 不是左标签系列

### 第2步：右侧是不是竖列表？

- 右侧为纵向排列的条目列表（上下滚动）→ 继续
- 横向排列/单区展示 → 考虑其他

### 第3步：顶部有没有贯穿全宽的标题栏？

- **无**贯穿全宽的T型标题栏 → **左标签+右竖列表** ✓
- **有**贯穿全宽的T型标题栏 → T型-上标题-左标签-右竖列表

---

## 常见混淆

### vs T型-上标题-左标签-右竖列表（最关键！）

| 特征 | 左标签+右竖列表 | T型-上标题-左标签-右竖列表 |
|------|--------------|--------------------------|
| 顶部标题 | **无贯穿全宽标题** | **有T型标题贯穿全宽** |
| 结构 | 二分（左标+右列） | T字（上横+下二分） |

---

## 判断口诀

> **左标竖立右列排，无横标题顶上盖。T型有杠此无杠，左标签系基础在。**
