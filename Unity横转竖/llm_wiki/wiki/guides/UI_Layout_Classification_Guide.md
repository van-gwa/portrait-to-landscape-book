## 一、什么是布局分类？

### 定义

**布局分类系统**是一种将 UI 界面按照规律进行分组和命名的方法。

**核心目标**：
- ✅ 让任何新 UI 都能快速归类
- ✅ 用简洁的名称描述复杂的布局
- ✅ 便于设计者和开发者的沟通

### 规模

```
分类覆盖：
├─ 布局类型：62 种
├─ UI 总数：910 个
└─ 覆盖率：100%（包括所有项目UI）
```

## 二、4 层分类维度详解

### 维度 1：主体结构（What）

**问题**：这个 UI 的整体形态是什么？

**三类结构**：

#### 弹框类 (4种, 141个UI)
```
┌──────────────────┐
│  确认弹框         │  ← 小浮窗，中心位置
│  ┌──────────────┤
│  │ 标题         │
│  │ 提示文字     │
│  │ [确认] [取消] │
│  └──────────────┘
└──────────────────┘

特征：
✓ 相对较小的窗口
✓ 居中显示
✓ 有明确标题
✓ 底部有操作按钮
✓ 背景有半透明遮罩
```

**细分类型**：
- 确认弹框 (87个) - 标题+文字+按钮
- 使用弹框 (24个) - 说明+配置+按钮
- 活动打脸弹窗 (30个) - 奖励展示

#### 全屏类 (5种, 32个UI)
```
┌──────────────────────────┐
│                          │
│   占满整个屏幕           │  ← 全屏，主界面
│                          │
│   精美背景+内容          │
│                          │
└──────────────────────────┘

特征：
✓ 占满整个视口
✓ 有精美背景图
✓ 作为主要交互界面
```

**细分类型**：
- 全屏背景界面 (12个) - 大背景+规则内容
- 全屏入口 (1个) - 全屏+单入口
- 全屏获得新奖励 (3个) - 奖励展示动画
- 登录加载全屏 (3个) - 登录/加载画面
- 灯笼标签+彩带+全屏 (2个) - 节日装饰

#### 面板/窗口类 (其余, ~737个UI)
```
┌──────────────────┐
│ 标题栏           │
│ ┌──────────────┤  ← 中等大小，独立窗口
│ │ 内容         │
│ │ ...         │
│ └──────────────┘
└──────────────────┘

特征：
✓ 中等大小的窗口
✓ 不占满整屏
✓ 独立的窗口框
✓ 规则的内容布局
```

### 维度 2：布局方向（How）

**问题**：UI 内部的内容如何分割和排列？

#### 左右布局 (15种, 166个UI)
```
┌─────┬─────────────┐
│左   │    右       │  ← 左边一个区域，右边一个区域
│ ↓   │            │
│列表 │  内容      │
│     │            │
└─────┴─────────────┘
```

**特征**：
- 左侧通常是列表、标签或导航
- 右侧是主要内容或列表
- 有明确的竖线分割

**细分类型**：
- 左-右 (36个) - 简单两列
- 左标签+右竖列表 (14个) - 左标签+右列表
- 左广告图-右列表 (13个) - 左大图+右列表
- 等等 (共15种)

#### 上下布局 (12种, 167个UI)
```
┌──────────────────┐
│  上（标题）      │  ← 顶部标题
├──────────────────┤
│  中（列表/内容）  │
│                  │
│  下（信息/按钮）  │  ← 底部操作
└──────────────────┘
```

**特征**：
- 纵向分为多个区域
- 通常是上标题+中内容+下操作
- 竖向滚动展示

**细分类型**：
- 上标题-中竖列表-下信息 (92个) ⭐⭐⭐ 最常见
- 上标题-中横列表-下信息 (57个) ⭐⭐
- 展示-横条-条内信息 (63个) - 多个横条
- 等等 (共12种)

#### T型布局 (2种, 54个UI)
```
┌──────────────────────┐
│  上（标题）          │  ← T的上：一条贯穿全宽的横线
├──────┬───────────────┤
│左    │    右         │  ← T的下：左右两列
│标签  │  列表         │
└──────┴───────────────┘
```

**特征**：
- 上方有一条贯穿全宽的标题/标签栏
- 这条线是全局分割，不仅仅属于右侧
- 下方分左右两部分
- **关键**：标题线的范围很重要！

**细分类型**：
- T型-上标题-左-右 (26个) - 简单T字
- T型-上标题-左标签-右竖列表 (28个) - T字+左标签+右竖列

**T型的常见错误**：
```
❌ 错误：把"上方标题"看成只属于右框的标题
✅ 正确：上方标题贯穿左右，是全局分割线

【实际案例】UI_BeautyTravel：
┌──────────────────────┐
│ 旅游与颜 [报名]      │  ← 这条线贯穿全宽！
├──────┬───────────────┤
│美妆  │♦ 女人选择    │  ← 现在知道这是T型结构了
│资料  │  选择女性    │
│...   │[去看看]      │
└──────┴───────────────┘
```

### 维度 3：装饰元素（Style）

**问题**：有没有特殊的装饰框或背景？

#### 木栏装饰 (6种, 38个UI)
```
╔════════════════════╗
║  [标签1] [标签2]   ║  ← 蒸笼标签（多个顶部标签）
╠════════════════════╣
║ 左   │ 中   │ 右  ║
║ 竖   │ 内   │ 信  ║
║ 列   │ 容   │ 息  ║
╚════════════════════╝
     ↑ 黄棕色木栏
```

**特征**：
- 四周有黄棕色的木栏框
- 古风游戏美术风格
- 通常有"蒸笼标签"（顶部多个标签分页）

**细分类型**：
- 四边木栏+左中右 (11个) - 左竖+中内容+右信息
- 四边木栏+横列表 (10个) - 木栏框+横列表
- 四边木栏底 (6个) - 木栏在底部
- 等等 (共6种)

#### 艺术设计装饰 (3种, 21个UI)
```
┌─────┐               ┌─────┐
│🎨  │  中间内容      │🎨  │  ← 彩色装饰框
│ 装  │               │ 装  │
│ 饰  │               │ 饰  │
│框  │               │框  │
└─────┘               └─────┘
  ↑                     ↑
左装饰                右装饰
```

**特征**：
- 左右两边有彩色装饰图案
- 中间是实际内容
- 美术风格突出

**细分类型**：
- 左右艺术设计+横列表 (3个)
- 左右艺术设计+竖列表 (3个)
- 左艺术设计+左-右 (14个)

#### 其他特殊装饰 (多种)
```
🏮 灯笼标签+彩带+全屏 (2个)
🎋 木屏风+横列表 (3个)  
🧱 墙+小旗+横列表 (3个)
```

#### 无特殊装饰 (大多数)
```
就是纯粹的布局，
没有特殊框或装饰。
最常见的情况！
```

### 维度 4：细节特征（Details）

**问题**：具体的排列和元素是什么？

#### 列表方向
- **竖列表** = 纵向排列，上下滚动
- **横列表** = 横向排列，左右滚动

#### 标题位置
- **上标题** = 顶部标题栏
- **左标签** = 左侧标签/分类

#### 标签类型
- **单标签** = 一个标题行
- **多标签页** = 多个可切换的标签（蒸笼标签）

#### 内容形式
- **竖条** = 竖向条形
- **横条** = 横向条形
- **格子** = 网格排列

## 三、判断流程（5步法）

### Step 1：看第一眼（第一层）
```
看整个窗口
├─ 小浮窗 → 弹框
├─ 全屏 → 全屏
└─ 中等窗口 → 面板
```

### Step 2：看内容分割（第二层）
```
内容如何排列
├─ 左右分割 → 左右布局
├─ 上中下分割 → 上下布局
├─ 上一线+下左右 → T型布局
└─ 其他 → 特殊布局
```

### Step 3：检查装饰（第三层）
```
有没有特殊框
├─ 黄棕色木栏 → 木栏装饰
├─ 左右彩色装饰 → 艺术装饰
├─ 灯笼/屏风/墙 → 特殊装饰
└─ 没有 → 无装饰
```

### Step 4：看细节（第四层）
```
具体的排列特征
├─ 列表方向：竖 vs 横
├─ 标题位置：上 vs 左
├─ 标签数量：单 vs 多
└─ 内容形式：条 vs 格子
```

### Step 5：对标现有类型
```
对照 62 种已知类型
└─ 找到最接近的那个
```

## 五、命名规律详解

### 规则

```
[装饰] + [方向] + [内容] + [位置]

每个部分的含义：
1. 装饰 - 有没有特殊框
2. 方向 - 如何分割
3. 内容 - 是什么内容
4. 位置 - 在哪个位置
```

### 例子

```
"四边木栏（蒸笼标签）+左中右"
├─ 装饰：四边木栏
├─ 修饰：蒸笼标签（多个顶部标签）
├─ 方向：左中右（三列）
└─ 说明：+号表示内含关系

"上标题-中竖列表-下信息"
├─ 上部分：标题
├─ 中部分：竖列表
├─ 下部分：信息
└─ 说明：-号表示纵向层次

"展示-横条-左图-右信息"
├─ 类型：展示
├─ 结构：横条形
├─ 左内容：图
├─ 右内容：信息
└─ 说明：在横条内分左右
```

## 六、常见错误和陷阱

### 🚨 陷阱1：混淆"竖列表"和"横列表"
```
❌ "有好几行就是竖列表"
✅ 竖列表 = 纵向滚动，项目竖向排列
✅ 横列表 = 横向排列，项目在同一行或水平滚动
```

### 🚨 陷阱2：忽视装饰元素
```
❌ "左右列表就是左-右布局"
✅ 如果有木栏框 = 四边木栏系列
✅ 如果有彩色装饰 = 艺术设计系列
```

### 🚨 陷阱3：T型 vs 左标签页
```
❌ 关键看上面的标题是否贯穿全宽
✅ 贯穿全宽 = T型
✅ 只在右框内 = 左标签页

【实际对比】
T型：
┌──────────────────┐
│ 标题（贯穿）      │ ← 这是全局分割线
├──────┬───────────┤
│标签  │ 内容      │

左标签页：
┌───┬──────────────┐
│   │ 标题(框内)   │ ← 只在右框
│标 │ ┌──────────┤
│签 │ │ 内容     │
│页 │ └──────────┘
└───┴──────────────┘
```

### 🚨 陷阱4：标题范围的理解
```
❌ "上面有标题就是上标题类"
✅ 关键是看标题的范围
- 贯穿全宽 = 全局标题（T型/上下型）
- 只在框内 = 局部标题（框内结构）
```

## 七、实战演练

### 案例1：UI_RankActServerRank
```
看图：上面有"医服排行"标题
     中间有竖列表（排名、名称、积分）
     下面有"本服w"按钮

判断：
Step 1: 中等窗口 → 面板
Step 2: 上中下纵向 → 上下布局
Step 3: 无特殊装饰 → 无装饰
Step 4: 上标题+竖列表+下操作 → 精确匹配

结论：上标题-中竖列表-下信息 ✓
```

### 案例2：UI_BeautyTravel
```
看图：上面"旅游与颜"贯穿全宽
     左边：美妆资料、美女名人等标签
     右边：♦女人选择衣的...等内容

判断：
Step 1: 中等窗口 → 面板
Step 2: 上一线贯穿+下左右 → T型
Step 3: 无特殊装饰 → 无装饰
Step 4: 左标签+右竖列表 → 精确匹配

结论：T型-上标题-左标签-右竖列表 ✓
```

## 八、应用场景

### 🎨 设计师
- 快速确定新 UI 的分类

- 保持设计一致性

- 理解 UI 的结构和层级

- 统计各类型 UI 的数量
- 评估设计工作量
- 规划设计系统

### 🧠 LLM/AI
- 理解 UI 的语义结构
- 生成代码或描述
- 进行 UI 分析和优化
