# 左标签页-右框内-上标题-中竖列表-下信息

## 定义

「左标签页-右框内-上标题-中竖列表-下信息」是左标签系列中最复杂的命名类型——**左侧竖标签切换 + 右侧框内包含完整的三段式（标题+竖列表+底部信息）**。共7个UI。

**与「T型-上标题-左标签-右竖列表」的核心区别**：本类型的标题**在右框内部**，不贯穿全宽。T型的标题**跨越左右全宽**形成T字横杠。

## 布局结构

```
┌──┬──────────────────────┐
│  │ [标题]  ← 在框内！  │  不是贯穿全宽！
│  ├──────────────────────┤
│标│  条目1               │
│签│  条目2               │  ← 框内竖列表
│页│  条目3               │
│  ├──────────────────────┤
│  │  底部信息/统计       │
└──┴──────────────────────┘
```

## 三步判断法（4样本+1验证，通过）

### 第1步：有没有左侧竖标签？

- 左侧有竖向标签切换 → 继续

### 第2步：右侧是不是独立框体？

- 右侧有一个明确边框的内容区 → 继续

### 第3步：框内标题贯穿全宽吗？

- 标题**在右框内部**，不贯穿左右全宽 → **左标签页-右框内-上标题-中竖列表-下信息** ✓
- 标题**贯穿左右全宽** → T型-上标题-左标签-右竖列表

---

## vs T型-上标题-左标签-右竖列表（最易混淆！）

| 特征 | 左标签页-右框内 | T型 |
|------|--------------|-----|
| 标题位置 | **右框内部** | **跨越全宽** |
| 结构逻辑 | 标+框(标+列+底) | T横杠 + 下二分 |

**一句话区分**：看标题是在框里还是框外！

---

## 判断口诀

> **左标右框两分清，标题只在框内存。不跨全宽无T杠，左标签页框内型。**
