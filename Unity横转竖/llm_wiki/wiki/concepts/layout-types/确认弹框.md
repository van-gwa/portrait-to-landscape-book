# 确认弹框

## 定义

「确认弹框」是弹框类中**数量最多（87个）、边界最模糊**的类型——全屏遮罩+居中弹窗+操作按钮。涵盖确认/取消双按钮、单操作按钮、设置配置、输入填写、购买确认、激活确认等多种子形态。

**核心判据（修正后）**：遮罩+居中弹窗+**含明确操作按钮**（单按钮或双按钮均可），目的是让用户做决策/确认/输入后执行操作。

## 布局结构

```
┌──────────────────────────────┐
│       全屏遮罩               │
│  ┌────────────────────┐      │
│  │ [标题]             │      │
│  │                    │      │
│  │ 内容区             │      │
│  │ (物品/提示/输入)   │      │
│  │                    │      │
│  │ [取消] [确认]      │      │  ← 双按钮 或 [激活/前往/开启]
│  └────────────────────┘      │
└──────────────────────────────┘
```

## 判断方法（全量87张补读后修正）

### 必中特征（~95%满足）

- 全屏黑色半透明遮罩
- 居中弹窗
- 古风纸质/羊皮纸边框
- 顶部标题

### 按钮形态（多样性大）

| 形态 | 占比 | 示例 |
|------|------|------|
| 取消+确认/确定 | ~40% | BagComposeTips, BeautyTalkTip |
| 单操作按钮(前往/激活/开启) | ~30% | TokensError, GoGuestTip |
| 确认+取消(非对称文字) | ~10% | ExpandAgentStop(结束/继续) |
| 输入框+确认 | ~5% | ChildRenameTip, RoleRenameTip |
| 功能设置+开启按钮 | ~5% | CyberIllusionAuto(自动模式设置) |

### ⚠️ 容易混淆的边界case

| UI名称 | 表象 | 归为确认弹框的原因 |
|--------|------|-------------------|
| UI_ChildMail | 写信输入界面 | 寄出=确认操作，含消耗信息 |
| UI_ExpandDispatchList | 派遣列表+一键处理 | 一键处理=批量确认 |
| UI_ExpandAkfReward | 领取奖励展示 | 领取=确认获得 |
| UI_Sidebar | 侧边栏 | 功能入口也是一种确认 |
| UI_TradingPortComposeQuick | 一键合成操作 | 勾选+合成=批量确认 |
| UI_BanquetOfFamilyOpen | 宴会开启 | 开启=确认创建 |

## vs 使用弹框

| 特征 | 确认弹框 | 使用弹框 |
|------|---------|---------|
| 数量选择器 | 罕见 | **常见** |
| 按钮 | 确认/取消 或 单操作 | 使用/合成/购买 |
| 场景 | 决策确认 | 物品消耗 |

**一句区分**: 有数量选择器→使用弹框；无→确认弹框

## vs 活动打脸弹窗

确认弹框更简洁，无角色立绘/倒计时/折扣标签。

## vs 功能开启&展示

功能开启&展示是宣告式，确认弹框是决策式。

## 判断口诀

> **遮罩居中题操作，取消确认双钮多。单钮前往也确认，无数量选即此科。**

## 主要特征（修正）

- 87个UI全满足：遮罩+居中弹窗+操作按钮
- 按钮形态多样：双按钮(~40%) > 单按钮(~30%) > 其他(~30%)
- 边界模糊：写信、设置、合成、领取奖励都被归入确认弹框
- 与使用弹框的最可靠区别：**有无数量选择器**
