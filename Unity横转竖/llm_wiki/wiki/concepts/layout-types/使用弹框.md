# 使用弹框

## 定义

「使用弹框」是道具/物品**消耗操作**的确认界面——全屏遮罩 + 居中弹窗 + 物品展示 + **数量选择/调节** + 底部使用按钮。属弹框类大类，共 24 个 UI。

**与确认弹框的核心区别**：使用弹框 = **消耗物品**（合成/购买/分解），确认弹框 = **确认决策**（是/否）。

## 布局结构

```
┌──────────────────────────────┐
│        全屏黑色遮罩          │
│  ┌────────────────────┐      │
│  │ 物品图标+名称      │      │
│  │ 拥有数量：XX       │      │
│  │                    │      │
│  │ 数量: [-] 10 [+]   │      │  ← 数量选择器（核心标志）
│  │                    │      │
│  │ 消耗/花费说明      │      │
│  │                    │      │
│  │ [    使用/合成    ]│      │  ← 操作按钮
│  └────────────────────┘      │
└──────────────────────────────┘
```

## 三步判断法（8样本+2验证，盲测2/2通过）

### 第1步：全屏遮罩 + 居中弹窗？

- 黑色遮罩背景 + 居中弹窗 → 继续
- 否 → 不是弹框类

### 第2步：有没有物品展示 + 数量调节？

- 界面中有**道具图标+名称+数量**，且提供数量调节（+/-按钮/滑块/最大化）→ 极大概率是使用弹框
- 无数量调节但有使用行为（分解/开启）→ 也可能是使用弹框

**数量选择器是核心判据**（约80%的UI都有）。

### 第3步：底部按钮是「使用/合成/购买/分解」？

- 按钮文字为动作性动词（使用/合成/购买/分解/开启）→ **使用弹框** ✓
- 按钮为确认/取消双按钮结构 → 考虑确认弹框

## 常见混淆类型及区分

### vs 确认弹框

| 特征 | 使用弹框 | 确认弹框 |
|------|---------|---------|
| **核心行为** | 消耗/使用物品 | 决策确认（是/否） |
| **数量选择器** | 常见（+/-调节） | 罕见 |
| **物品展示** | 有（道具图标+名称） | 不一定 |
| **底部按钮** | 使用/合成/购买/分解 | 确认+取消 / 单确认 |
| **典型场景** | 使用道具、合成装备、购买商品 | 是否删除、是否开启 |

### vs 活动打脸弹窗

| 特征 | 使用弹框 | 活动打脸弹窗 |
|------|---------|-------------|
| 目的 | 物品消耗 | 营销转化 |
| 数量选择 | 有 | 无 |
| 装饰 | 简洁 | 立绘+倒计时+折扣 |

## 判断口诀

> **黑底居中道具显，加减数量随你选。使用合成或分解，物品消耗即此弹。**

## 主要特征

- 全屏黑色遮罩 + 居中弹窗（100%）
- 数量选择器是核心标志（约80%有，+/-按钮/最大化）
- 底部为消耗性操作按钮（使用/合成/购买/分解）
- 与确认弹框的区别：物品消耗 vs 决策确认
