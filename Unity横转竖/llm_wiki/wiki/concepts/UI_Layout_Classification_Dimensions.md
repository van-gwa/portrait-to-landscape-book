# UI 布局分类的 4 层维度

## 定义

**UI 布局分类系统**将所有 UI 界面按照 4 个递进的维度进行分类，从而能够快速、准确地识别任何新 UI 属于哪种布局类型。

这个分类体系覆盖了 **62 种布局类型**，涵盖 **910 个实际 UI**。

---

## 4 层分类维度

### 第一层：主体结构（决定基本形态）

**定义**：UI 整体的形态和定位方式

**分类**：
- **弹框类** (4种) - 小的浮窗，居中，用于临时交互
- **全屏类** (5种) - 占满整个屏幕，作为主要界面
- **面板/窗口类** (多种) - 中等大小的独立窗口

**识别方法**：
```
看窗口大小和位置
├─ 小浮窗 → 弹框类
├─ 占满屏幕 → 全屏类
└─ 中等窗口 → 面板/窗口类
```

**统计**：
- 弹框类：4种，141个UI (15.5%)
- 全屏类：5种，32个UI (3.5%)
- 面板类：其余，737个UI (81%)

---

### 第二层：布局方向（内容如何分割）

**定义**：UI 内部的主要分割方式和流向

**主要方向**：
- **左右布局** (15种) - 左右两列或多列分割
- **上下布局** (12种) - 上中下多行分割
- **T型布局** (2种) - 上一行+下左右分割
- **其他复杂** (多种) - 左标签页+右框、多列等

**识别方法**：
```
看内容如何排列
├─ 左侧列表/标签 + 右侧内容 → 左右布局
├─ 上标题 + 中内容 + 下信息 → 上下布局
├─ 上标题贯穿全宽 + 下左右分割 → T型布局
└─ 其他复杂组合 → 其他
```

**统计**：
- 左右布局：15种，166个UI (18%)
- 上下布局：12种，167个UI (18%)
- T型布局：2种，54个UI (6%)
- 其他：多种，523个UI (58%)

---

### 第三层：装饰元素（美术设计风格）

**定义**：UI 边框、装饰框、背景等美术设计元素

**主要装饰类型**：
- **木栏装饰** (6种) - 古风黄棕色木栏框
- **艺术设计** (3种) - 左右彩色装饰边框
- **其他特殊** (多种) - 灯笼、屏风、墙等
- **无特殊装饰** (大多数) - 纯布局，无特殊装饰

**识别方法**：
```
看是否有特殊装饰元素
├─ 四周黄棕色木栏 → 木栏装饰
├─ 左右彩色装饰框 → 艺术设计
├─ 特殊图案（灯笼/屏风/墙） → 特殊装饰
└─ 无特殊框 → 无装饰
```

**统计**：
- 木栏装饰：6种，38个UI (4%)
- 艺术设计：3种，21个UI (2%)
- 其他特殊：5种，20个UI (2%)
- 无装饰：大多数，831个UI (92%)

**木栏装饰的特征**：
```
【蒸笼标签】
├─ 上面多个标签分页（像蒸笼）
├─ 四周黄棕色木栏框
└─ 古风游戏美术风格

【内部结构】
├─ 左竖列表 + 中内容 + 右信息
├─ 横向列表
├─ 网格排列
└─ 底部框
```

---

### 第四层：细节特征（精确匹配类型）

**定义**：在确定了前三层之后，用细节特征进行精确匹配

**常见细节特征**：
- **列表方向**：竖列表 vs 横列表
- **标题位置**：上方、左上、框内等
- **标签类型**：单标签 vs 多标签页
- **内容排列**：竖条、横条、格子等
- **附加元素**：返回按钮、关闭按钮等

**示例**：
```
布局类型：T型-上标题-左标签-右竖列表
├─ 第一层：面板类
├─ 第二层：T型布局（上一横线 + 下左右）
├─ 第三层：无特殊装饰
└─ 第四层：
   ├─ 上面：一条贯穿全宽的标题
   ├─ 左下：竖向标签列表
   └─ 右下：竖向内容列表


布局类型：四边木栏（蒸笼标签）+左中右
├─ 第一层：面板类
├─ 第二层：左右布局（左中右三列）
├─ 第三层：木栏装饰（蒸笼标签）
└─ 第四层：
   ├─ 有多个顶部标签分页
   ├─ 左：竖列表
   ├─ 中：主内容区
   └─ 右：信息区
```

## 分类的层级关系

```
┌─────────────────────────────────────┐
│   第一层：主体结构                    │
│   弹框 / 全屏 / 面板                │
└─────────────────────────────────────┘
           ↓
┌─────────────────────────────────────┐
│   第二层：布局方向                    │
│   左右 / 上下 / T型 / 其他           │
└─────────────────────────────────────┘
           ↓
┌─────────────────────────────────────┐
│   第三层：装饰元素                    │
│   木栏 / 艺术设计 / 特殊 / 无        │
└─────────────────────────────────────┘
           ↓
┌─────────────────────────────────────┐
│   第四层：细节特征                    │
│   列表方向 / 标题位置 / 标签类型等    │
└─────────────────────────────────────┘
           ↓
    🎯 确定确切的布局类型
```

## 判断流程

### 三步快速判断法

```
第一眼看 → 第一层（大小位置）
   ↓
仔细看 → 第二层（内容分割）
   ↓
检查细节 → 第三层+第四层（装饰和特征）
   ↓
✅ 确定布局类型
```

### 每一层的关键问题

**第一层**：
- 这个窗口有多大？
- 是小浮窗、全屏还是中等窗口？

**第二层**：
- 内容如何分割？
- 是左右分割、上下分割还是其他？

**第三层**：
- 有没有特殊的装饰框？
- 是木栏、艺术框还是没有？

**第四层**：
- 具体的列表方向是什么？
- 标题和标签在哪里？
- 有什么特殊的附加元素？

## 命名规律

### 为什么布局类型的名称这样起？

```
命名模式：[装饰元素] + [主体方向] + [内容类型] + [位置/细节]

示例解读：
═════════════════════════════════════

"四边木栏（蒸笼标签）+左中右"
├─ 装饰元素：四边木栏
├─ 修饰词：蒸笼标签（多个顶部标签）
├─ 主体方向：左中右（三列）
└─ 说明：+号表示组合关系


"上标题-中竖列表-下信息"
├─ 第一部分：上标题
├─ 第二部分：中竖列表
├─ 第三部分：下信息
└─ 说明：-号表示上中下的纵向关系


"展示-横条-左图-右信息"
├─ 类型：展示
├─ 结构：横条形
├─ 内容：左图+右信息
└─ 说明：细化描述内容排列

"左标签页-右框内-上标题-中竖列表-下信息"
├─ 左部分：标签页（导航）
├─ 右部分：框内
│   ├─ 上：标题
│   ├─ 中：竖列表
│   └─ 下：信息
└─ 说明：嵌套结构用-连接
```

### 常用词汇含义

| 词汇 | 含义 |
|-----|------|
| **竖列表** | 纵向排列，上下滚动 |
| **横列表** | 横向排列，左右滚动 |
| **标签** | 分类标签/标题栏 |
| **标题** | 顶部标题行 |
| **信息** | 底部信息区/按钮区 |
| **框** | 边框/容器/框体 |
| **展示** | 显示/呈现类 |
| **条** | 条形/条纹元素 |
| **+** | 并列关系 |
| **-** | 上下左右关系 |

## 分类的优势

### 为什么要用 4 层维度？

✅ **易于学习**
- 不需要记住全部 62 种
- 掌握 4 个维度就能推导出大多数类型

✅ **快速判断**
- 按顺序问 4 个问题就能确定类型
- 避免被细节迷惑

✅ **系统完整**
- 覆盖所有 910 个 UI
- 无遗漏、无重复

✅ **易于扩展**
- 新的 UI 出现时，按 4 层维度判断即可
- 自动融入现有分类系统
