# UI 布局

## 定义
**UI 布局**是 Unity 游戏项目中用户界面的空间组织方式。它定义了界面元素（按钮、列表、文本、图像等）在屏幕上的相对位置和尺寸关系。

## 核心原理

### 布局的作用
- **视觉组织**：让界面内容有序排列
- **交互指导**：通过空间关系引导用户操作
- **响应式适配**：支持不同屏幕分辨率和方向

### 布局的维度
```
维度 1：结构维度
  ├─ 单列布局（Single Column）
  ├─ 多列布局（Multi-Column）
  ├─ 网格布局（Grid）
  └─ 自由浮动（Floating）

维度 2：内容维度
  ├─ 标题区
  ├─ 内容区（列表、网格、详情）
  ├─ 操作区（按钮、输入框）
  └─ 信息区（页脚、状态栏）

维度 3：装饰维度
  ├─ 边框装饰（Frame Border）
  ├─ 背景装饰（Background）
  ├─ 分隔符
  └─ 特殊元素（标签、徽章）
```

## 布局分类方法

### 方法 1：按内容流向
- **竖向流布局**：内容从上到下排列
- **横向流布局**：内容从左到右排列
- **混合流布局**：结合竖向和横向

### 方法 2：按框架类型
- **四边框架**：上下左右都有边框
- **局部框架**：仅部分边框
- **无框架**：仅背景色

### 方法 3：按列表类型
- **竖向列表**：垂直滚动
- **横向列表**：水平滚动
- **网格列表**：二维排列

## 横竖屏适配

当 UI 从竖屏模式转换到横屏模式时，布局需要重新调整：

### 竖屏特点（Portrait）
```
┌─────────┐
│  标题   │
├─────────┤
│  内容   │
│  列表   │
│  ...    │
├─────────┤
│ 按钮    │
└─────────┘
```
- 宽度受限
- 高度充足，可以纵向扩展
- 列表通常为单列

### 横屏特点（Landscape）
```
┌──────────────────┐
│标题              │
├────────┬─────────┤
│内容    │ 信息    │
│列表    │ 详情    │
│        │ ...     │
├────────┴─────────┤
│    按钮          │
└──────────────────┘
```
- 宽度充足
- 高度受限
- 可以放多列内容
