## 问题管理流程

### 发现问题

当检查任务发现不一致时：

1. **记录基本信息**
   - UI 名称
   - 发现时间
   - 分析结果
   - 已有分类

2. **分析问题原因**
   - 分类错误？
   - 缺少分类？
   - 其他原因？

3. **判断优先级**
   - 高：影响多个 UI 的分类系统
   - 中：单个 UI 的分类错误
   - 低：边界情况，可缓处理

### 处理问题

1. **根本原因分析**
   - 定位到具体的错误点
   - 确定正确的分类

2. **执行修正**
   - 更新相关的 Wiki 页面
   - 修正分类记录
   - 记录修复过程

3. **验证修复**
   - 再次检查确认修复成功
   - 更新问题状态为"已修复"

### 预防措施

- 完善识别指南
- 改进自动化检查的准确性
- 定期审查分类规则
