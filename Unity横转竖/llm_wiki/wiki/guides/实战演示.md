# 上标题-中竖列表-下信息

## 定义
这是一个**三区域布局**，将界面分为：
- **上区域**：标题和导航信息
- **中区域**：竖向滚动列表（主要内容）
- **下区域**：操作按钮或额外信息

## 布局示意图
```
┌─────────────────┐
│   标题区域      │
├─────────────────┤
│                 │
│  竖向列表       │
│  内容区域       │
│  (可滚动)       │
│                 │
├─────────────────┤
│  按钮/信息区    │
└─────────────────┘
```

## 特点
- **适应屏幕**：竖屏时高效利用竖向空间
- **交互清晰**：用户操作流程明确
- **内容优先**：列表区域占最大空间

## 应用场景
- 排行榜 (UI_RankActServerRank)
- 奖励展示 (UI_BanquetPopularityReward)
- 任务列表 (UI_TaskGroupReward)
- 记录列表 (UI_BanquetRecord)

## 包含的 UI 数量
**92 个** UI 使用此布局（最常见的布局类型）

## 相关布局
- [[concept/four-border-left-center-right]] - 用了框架装饰的变种
- [[concept/vertical-list]] - 核心元素：竖向列表

## 数据统计
- 平均 UI 复杂度：中等（8-12 个组件）
- 最常用的组件：Button, Text, ScrollRect
- 常见的装饰：四边框架、背景色

## 最近发现
- 2026-05-04：确认为最常见的布局类型
- 适合进一步细化成子类型（如有/无框架）

## 横屏适配建议
详见：[[llm_wiki/wiki/guides/Landscape_Adaptation_Guide|横屏适配指南]]（待创建）

  ├─ 横屏特点：宽度充足，高度受限
  └─ 调整方案：
     ├─ 列表改为双列或并排
     ├─ 信息区可以侧边
     └─ 标题可以更紧凑

相关资源：
  ├─ 当前排行榜 UI：[[UI_RankActServerRank]]

### ✅ 第 4 步：生成答案

**给用户的回答**（简化版）：

> 排行榜 UI 目前使用"上标题-中竖列表-下信息"布局 ([[concept/top-title-middle-list-bottom-info]])。
>
> 转换为横屏时的建议：
> 1. **列表调整**：从竖向单列改为竖向双列或网格，充分利用水平空间
> 2. **区域重排**：考虑将信息区改为侧边栏，而非底部栏
> 3. **标题优化**：标题可以更紧凑，节省竖向空间
>
> 详见：[[llm_wiki/wiki/guides/Landscape_Adaptation_Guide]]（详细指南，待更新）

## 🎓 学到的要点

### ✨ Ingest 工作流的价值
- 系统地处理大量新数据（不会遗漏）
- 自动建立概念和数据源之间的链接
- 保持 Wiki 的结构化和可导航性

### ✨ Query 工作流的价值
- 回答问题的同时充实知识库
- 发现 Wiki 的缺口和改进点
- 让每次问题都成为知识库进化的机会

### ✨ 组合效果
- Ingest + Query 形成良性循环
- Wiki 随着使用而不断演进
- 维护成本随规模稳定（不会线性增长）
