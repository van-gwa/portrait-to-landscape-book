# T型-上标题-左标签-右竖列表

## 定义

该布局通过**T型-上标题-左标签-右竖列表**的空间组织方式，实现内容的有效展示和交互。属 T型布局 大类，共 28 个 UI。

## 布局结构

```
┌──────────────────────┐
│      标题            │  ← T的横杠：贯穿全宽的水平分割区域
├────────┬─────────────┤
│ 标签   │  [Item 1]  │
│ ┌───┐  │  [Item 2]  │ ↕ 可竖向滚动
│ │A  │  │  [Item 3]  │
│ ├───┤  │  ...       │
│ │B  │  │            │
│ ├───┤  │            │
│ │C  │  │            │
│ └───┘  │            │
└────────┴─────────────┘
  ↑        ↑
 T的下左  T的下右
```

## 三步判断法（基于28个UI图片分析）

### 第1步：找"T的横杠"——顶部是否有贯穿全宽的水平分割？

这是**最关键的判断依据**，也是区分 T型 与 左标签页 的核心。

**看顶部**：是否存在一条"从上到下第一道水平分割线"，且这条线跨越了界面的**左右全宽**？

有三种常见形态：

| 形态 | 特征 | 典型例子 |
|------|------|----------|
| **装饰横幅型** | 贯穿全宽的装饰性Banner，含角色立绘、活动标题、倒计时等 | UI_TaskSprint、UI_ActStrengthSpurt、UI_AliMinGameClub、UI_ActMoonSpurt |
| **标题字+底纹型** | 居中标题文字，但在视觉上形成一道横跨全宽的分割带 | UI_BeautyTravel（"结伴同游"）、UI_TeamList（"组队列表"） |
| **功能条+标题型** | 顶部资源条/返回按钮等功能元素与标题共同构成横向分割 | UI_GuildWarTask、UI_RankActTask、UI_MarketAdvanced |

**关键判据**：这条水平分割线是"全局"的——它把整个界面切成"上"和"下"两部分，下面的部分才进行左右分栏。如果标题只出现在右侧框内部、不跨越左侧标签区，则不是 T型。

### 第2步：看"T的下左"——左侧是否有竖向标签列表？

确认左侧区域的特征：

- **排列方式**：竖向堆叠的按钮/标签，通常 2~5 个
- **宽度**：窄列，约占界面总宽的 1/5 ~ 1/4
- **功能**：用于分类切换，切换后右侧内容更新
- **形态**：矩形的Tab按钮是最常见形式；少数使用圆形图标按钮（如 UI_ExpandAkfPreview）

所有28个UI均满足此特征。

### 第3步：看"T的下右"——右侧是否为竖列表？

确认右侧主内容区：

- **列表方向**：纵向排列的条目列表（竖向滚动）
- **条目形态**：每个条目通常是横向的卡片/行，包含图标、文字、进度条、按钮等
- **常见变体**：
  - 任务列表（带进度条+操作按钮）：最典型，如 UI_BattlePassTask、UI_StreetFightTask
  - 排行榜列表（带排名+数据）：如 UI_GuildWarRank、UI_GuildChallengeContrib
  - 详情展示（单条信息卡）：如 UI_RankActGift、UI_InvestmentSkill
  - 空态占位（插画+提示文字）：如 UI_TeamList

## 常见混淆类型及区分

### 🚨 最易混淆：T型-上标题-左标签-右竖列表 vs 左标签页-右框内

```
【T型-上标题-左标签-右竖列表】          【左标签页-右框内】
┌──────────────────────┐                ┌───┬─────────────────┐
│    标题（全局）      │ ← 关键！       │   │  标题（框内）   │ ← 只在右框
├──────┬───────────────┤                │   │ ┌─────────────┤
│标签A │  内容列表     │                │标 │ │  内容       │
│标签B │              │                │签 │ │             │
│标签C │              │                │页 │ └─────────────┘
└──────┴───────────────┘                └───┴─────────────────┘

判断核心：标题/顶部分割是否覆盖左标签+右内容两者的全宽？
  是 → T型 ✓
  否 → 左标签页
```

### 与 T型-上标题-左-右 区分

T型-上标题-左-右 的右侧不是竖列表，而是与左边对称的内容区域（如详情面板、图片等）。

### 与 上标题-中竖列表-下信息 区分

上标题-中竖列表-下信息 没有左侧标签栏，内容直接从上到下排列，不存在左右分割。

## 核心判断口诀

> **一横贯穿顶，二竖分左右，左标右列表，三者齐备即T型。**

1. **一横**：顶部有贯穿全宽的水平分割（标题/横幅/功能条）
2. **二竖**：下方被垂直分割为左右两栏
3. **左标**：左侧是竖向排列的标签/按钮（2~5个）
4. **右列**：右侧是纵向滚动的列表内容
5. **关键**：顶部的"一横"必须在"二竖"**之上**，同时覆盖左右两栏

---

## 主要特征

- 清晰的三区空间分区（上/左下/右下）
- 顶部区域贯穿全宽，是全局分割线
- 左侧标签用于分类切换，右侧内容随之更新
- 右侧为竖列表，支持纵向滚动
- T型布局类型的特征表现

## 横竖屏适配要点

- 布局在横竖屏转换时需要调整各区域的宽高比
- 关键信息应优先保证可见性
- 可根据屏幕方向调整内容的流向方式
