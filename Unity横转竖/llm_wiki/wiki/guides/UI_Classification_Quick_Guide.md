# UI 布局类型快速识别指南（避免认错）

## 核心原则

**三步验证法**：
1. **自动化工具初筛** → classify-ui-type 脚本快速分类
2. **特征对比确认** → 对照关键特征表，排除易混淆类型
3. **人工目视验证** → 在 Unity Editor 中二次确认

---

## 四边木栏系列的区分

### 🎯 最易混淆的三对类型

#### 对比 1：「四边木栏+左中右」vs「四边木栏+左右-宽列表」

| 维度 | 四边木栏+左中右 | 四边木栏+左右-宽列表 |
|------|--------|--------|
| **四边框** | ✅ 有 | ✅ 有 |
| **左侧** | 信息展示区（静态） | 🔴 **标签页/Tab**（切换导航） |
| **右侧** | 信息展示区（静态） | 🔴 **宽列表**（W≥1000，可滚动） |
| **列表特征** | ❌ 无列表 | ✅ 有宽列表 |
| **参考 UI** | UI_RankGobal | **UI_GachaCoinShop** |

**快速判断**：
- 看右侧有没有 **ScrollRect/List 容器**？如果有 → 是「左右-宽列表」
- 看左侧是不是 **Toggle/Tab 组件**？如果是 → 是「左右-宽列表」

---

#### 对比 2：「左边标签页+右边长文本列表」vs「四边木栏+左右-宽列表」

| 维度 | 左边标签页+右边长文本列表 | 四边木栏+左右-宽列表 |
|------|--------|--------|
| **四边框** | ❌ 无 | ✅ **有** |
| **左侧** | 标签页/Tab | 标签页/Tab |
| **右侧** | 纯文本列表（聊天/公告） | 🔴 **物品卡片列表** |
| **列表类型** | 单列、文本行、行高一致 | 多列、商品卡、图+文+价 |
| **参考 UI** | UI_Notice | UI_GachaCoinShop |

**快速判断**：
- 看有没有 **四边木栏**？没有 → 是「左标签页+右文本列表」
- 看右侧内容是什么？纯文本 → 文本列表；图标卡片 → 宽列表

---

#### 对比 3：「大背景+宽列表」vs「四边木栏+左右-宽列表」

| 维度 | 大背景+宽列表 | 四边木栏+左右-宽列表 |
|------|--------|--------|
| **四边框** | ❌ 无 | ✅ **有** |
| **左侧** | ❌ 无左侧结构 | ✅ **标签导航** |
| **背景** | 大背景（W≥800）| 大背景（可选） |
| **列表** | 宽列表（无标签页） | 宽列表（配合标签页） |
| **参考 UI** | 未发现 | UI_GachaCoinShop |

**快速判断**：
- 看有没有 **左侧标签页**？有 → 是「左右-宽列表」
- 看有没有 **四边框**？有 → 是「左右-宽列表」

## 自动化工具的使用

### ✅ 完整工作流（推荐）

```bash
# Step 1: 检测尺寸（排除"小尺寸缩放调整"）
calc-prefab-size <prefab_path>
# 输出: W=1237.55（> 1040，继续）

# Step 2: 自动分类
classify-ui-type <prefab_path>
# 输出: 四边木栏+左右-宽列表 ✔
#       节点分布：左=7个，中=6个，右=1个
#       列表宽度：1019.16 ≥ 背景×0.8 ✔

# Step 3: 人工确认
# 在 Unity Editor 中打开预制体
#  → 看节点结构，确认左标签页和右宽列表 ✔
```

### 📊 脚本输出的关键信息

分类脚本会输出：

```
═══════════════════════════════════════
  UI 界面类型分类报告 — UI_GachaCoinShop
═══════════════════════════════════════
① 木栏包边检测:      有四边木栏 ✔（上✔ 下✔ 左✔ 右✔）
② 有效节点数:       14 个（左区7个 中区6个 右区1个）
③ 关键组件:         TglGroupFun(标签页) ListGoods(列表)
④ 列表宽度:         1019.16 ≥ 背景×0.8 ✔
⑤ 形象展示:         无 ❌
⑥ 标题块:          无 ❌
═══════════════════════════════════════
【界面类型】四边木栏+左右-宽列表
  有四边木栏 + 左标签页 + 右宽列表
═══════════════════════════════════════
```

**关键读数**：
- ✅ 木栏包边 = 四边都有
- ✅ 左区 = 标签页或导航
- ✅ 右区 = 宽列表（W > 1000）
- ❌ 无形象、无标题 = 排除其他类型

## 手工检验清单

当自动化工具输出后，用这个清单二次确认：

### 📋 识别「四边木栏+左右-宽列表」的 4 大标志

- [ ] **木栏包边检验**
  - [ ] 节点树中有 4 个 Image 元素（上下左右各一个）
  - [ ] 都引用 `atlas_common_static.png` 的边框资源
  - [ ] 脚本输出显示「四边: 上✔ 下✔ 左✔ 右✔」

- [ ] **左侧标签页检验**
  - [ ] 节点名包含「TglGroupFun」或「ToggleGroup」
  - [ ] 有多个 Toggle 子元素（商品分类选项）
  - [ ] 宽度通常 150-250px，位置在最左侧
  - [ ] ❌ 不是静态的信息展示框

- [ ] **右侧宽列表检验**
  - [ ] 节点名包含「ListGoods」或「ScrollRect」或「List」
  - [ ] 宽度 ≥ 1000px（占背景宽的 80%+）
  - [ ] 内部有多个相同的商品卡片子节点
  - [ ] 支持纵向滚动（ScrollRect with Vertical）

- [ ] **排除其他类型检验**
  - [ ] ❌ 没有立绘/角色形象区（不是「宽图」类型）
  - [ ] ❌ 没有独立的标题块（不是「标题+」类型）
  - [ ] ❌ 右侧不是纯文本列表（不是「文本列表」类型）

## 常见错误案例

### ❌ 错误案例 1：把「左右-宽列表」看成「左中右」

**症状**：
```
看到：四边框 + 左侧+右侧 = 以为是"左中右"的三区布局
```

**错误原因**：
- 没有看清左侧是否有**导航标签页**
- 没有测量右侧列表的**宽度**（> 1000 才算宽列表）
- 混淆了"对等分区"和"导航+内容"的区别

**正确做法**：
```
1. 检查左侧：是 Tag/Toggle？还是静态信息框？
   → 如果是 Tag/Toggle = 导航，不是对等分区
2. 测量右侧宽度：> 1000？
   → 是 = 宽列表，不是小的信息区
3. 结论：导航+宽列表 = 「左右-宽列表」✔
```

---

### ❌ 错误案例 2：把「左右-宽列表」看成「左文本列表」

**症状**：
```
看到：左标签页 + 右列表 = 以为是"左标签页+右文本列表"
```

**错误原因**：
- 没有检查**四边框的有无**
- 没有看清右侧列表是**文本**还是**物品卡片**

**正确做法**：
```
1. 检查四边框：有没有装饰框？
   → 有 = 「左右-宽列表」
   → 没有 = 「左标签页+右文本列表」
2. 如果有四边框，再看右侧内容
   → 商品/物品卡片 = 宽列表 ✔
   → 纯文本行 = 文本列表 ✔
```

---

### ❌ 错误案例 3：把「左右-宽列表」看成「大背景+列表」

**症状**：
```
看到：大背景 + 列表 = 以为是"大背景+列表"
```

**错误原因**：
- 没有检查**左侧导航**的有无
- 没有检查**四边框**的有无

**正确做法**：
```
1. 检查左侧：有没有标签页？
   → 有 = 「左右-宽列表」✔
   → 无 = 可能是「大背景+列表」
2. 检查四边框：有没有？
   → 有 = 「左右-宽列表」✔
   → 无 = 「大背景+列表」
```

## 工作流建议

### 📌 日常分类步骤

```
【收到新 UI】
  ↓
【第 1 步】运行 classify-ui-type 脚本 (自动分类)
  ↓
【第 2 步】对比输出的"节点分布特征"
  → 左侧节点类型是什么？
  → 右侧有没有宽列表？
  → 有没有四边框？
  ↓
【第 3 步】查看对应的布局类型 Wiki 页面
  → 检查标准特征是否都符合
  → 对比"易混淆"的其他类型
  ↓
【第 4 步】在 Unity 中目视确认
  → 打开预制体
  → 点击各个关键节点，验证
  ↓
【完成】：发送分类结果
```

## 快速查询表

遇到不确定，马上查这个表：

### 有四边框的情况

| 左侧 | 右侧 | 类型 | 确认方式 |
|------|------|------|--------|
| 标签页 | 宽列表（W>1000） | **左右-宽列表** ⭐ | 右侧宽度 + 列表类型 |
| 标签页 | 宽图（立绘） | 左右-宽图 | 右侧是形象还是列表 |
| 信息展 | 信息展 | 左中右 | 左右都是静态，无列表 |
| 无左侧 | 宽列表 | 无四边框时才成立 | 有框 → 排除此项 |

### 无四边框的情况

| 左侧 | 右侧 | 类型 | 确认方式 |
|------|------|------|--------|
| 标签页 | 纯文本列表 | **左标签页+右文本列表** | 有框=错，无框+文本=对 |
| 无 | 宽列表 | 大背景+列表 | 有框=错，无框+列表=对 |
| 无 | 无 | 其他 | 查布局类型索引 |
