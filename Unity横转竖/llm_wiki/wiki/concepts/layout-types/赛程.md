# 赛程

## 定义

该布局以**淘汰赛对阵图（Bracket）**为核心内容组织方式，展示赛事进度、对阵关系和晋级路径。树形分支结构从两侧向中央汇聚，中央为冠军位。

## 布局结构

```
┌──────────────────────────────────────────┐
│  返回  [标题]  [?]  [数字]              │  ← 上：标题栏
├──────────────────────────────────────────┤
│                                          │
│  左4 ──┐                    ┌── 右4    │
│        ├──┐              ┌──┤          │
│           ├──┐      ┌──┤              │  ← 中：对阵图主体
│              ├──┬──┤                  │     (树形分支，向中央汇聚)
│             [冠军位]                   │
│  "我在这" →  "已淘汰"                 │
│  半决赛 info     决赛 info            │
│                                          │
├──────────────────────────────────────────┤
│  [赛区/阶段选择器 ▼]                    │  ← 下：筛选 + 状态 + 按钮
│  状态文字                              │
│  [按钮1] [按钮2] [按钮3] [按钮4]       │
└──────────────────────────────────────────┘
```

## 主要特征

### 核心视觉特征（基于图片分析）
- **对阵图树形结构**：左右两侧各 N 个初始位置，通过分支线向中央逐级汇聚，形成淘汰树
- **冠军位在中央**：通常为金色/橙色卷轴牌匾形式，是整个界面的视觉焦点
- **"我在这"提示**：标示当前玩家在赛程中的位置
- **"已淘汰"标记**：红色印章样式，标记淘汰状态
- **赛区/阶段切换**：底部下拉选择器，可切换不同赛区和赛程阶段

### 与近似类型的关键区分

| 对比维度 | 赛程 | 横条-VS | 战斗 |
|---------|------|---------|------|
| 核心结构 | 淘汰树/对阵图 | 横向 VS 条 | 战斗场景 |
| 宽高比 | ~2.2（很宽） | ~1.9 | ~1.6 |
| 平均宽度 | ~1356 | ~453 | ~1152 |
| 平均节点数 | ~58 | ~15 | ~86 |
| 冠军位 | ✅ 有 | ❌ 无 | ❌ 无 |
| 赛区选择器 | ✅ 有 | ❌ 无 | ❌ 无 |

### 判断流程（5 步法适配）

```
Step 1: 确认是中等面板窗口（非弹框、非全屏）
Step 2: 看内容 → 是否有树形对阵图？（左右分支向中央汇聚）
  ├─ YES → 继续 ↓
  └─ NO  → 排除「赛程」，检查横条-VS 或战斗
Step 3: 是否有冠军位（中央卷轴牌匾）？
  ├─ YES → 高度肯定是赛程
  └─ NO  → 仍可能是赛程（接力赛等变体），检查其他特征
Step 4: 底部是否有赛区/阶段选择器和功能按钮？
Step 5: 核对结构特征数据是否落在赛程范围（宽 ~1330-1390, 高 ~540-670）
```

## 横竖屏适配要点

- 对阵图的树形分支在横屏时沿 X 轴展开，竖屏时需沿 Y 轴重新布局或压缩
- 冠军位应始终保持在视觉中心
- 底部选择器和按钮行在竖屏时可能需要折叠或缩小
- 关键时间信息（半决赛/决赛时间）优先保证可见
