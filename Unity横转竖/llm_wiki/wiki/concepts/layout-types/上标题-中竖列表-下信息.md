# 上标题-中竖列表-下信息

## 定义

该布局通过**上标题-中竖列表-下信息**的三段式纵向组织，实现条目的纵向浏览与操作。属上标题系列大类，共 92 个 UI，是排名第2的高频类型。

**命名含义**：
- **上标题**：顶部居中标题（+返回+资源）
- **中竖列表**：中部为**垂直**排列的条目列表（上下滚动），条目为横向长条
- **下信息**：底部为个人信息/统计/操作按钮

**与「中横列表」的核心区别**：竖列表 = 条目纵向堆叠，上下滚动；横列表 = 条目横向排列，左右滚动。

## 布局结构

```
┌──────────────────────────────┐
│ [返回]    标题文字    [资源] │  ← 上：标题栏
├──────────────────────────────┤
│ ┌──────────────────────────┐ │
│ │ 条目1 (横向长条)        │ │
│ ├──────────────────────────┤ │
│ │ 条目2                   │ │  ← 中：竖列表
│ ├──────────────────────────┤ │     (上下滚动)
│ │ 条目3                   │ │
│ ├──────────────────────────┤ │
│ │ ...                     │ │
│ └──────────────────────────┘ │
├──────────────────────────────┤
│  个人统计/结算时间/[按钮]    │  ← 下：信息栏
└──────────────────────────────┘
```

## 三步判断法（基于92个UI图片分析）

### 第1步：看顶部有没有标题栏？

- **有居中标题**（如"区服排行""赛季任务""人气榜"等）→ 继续
- 无标题 → 考虑其他类型

### 第2步：看中部是不是**竖向**列表？

这是与「中横列表」的关键区分点：

| 特征 | 中竖列表（本类型） | 中横列表 |
|------|-------------------|----------|
| 条目排列 | **垂直堆叠**，上下滚动 | 水平排列，左右滚动 |
| 条目形状 | **横向长条**（占满宽度） | 竖向卡片 |
| 典型场景 | 排行榜、任务列表、记录 | 角色选择、商品选择 |

**判断关键**：条目是否从上到下罗列，且每个条目是一条横向长条？

### 第3步：看底部有没有信息/按钮栏？

- 有底部信息区（个人排名、倒计时、操作按钮） → **上标题-中竖列表-下信息** ✓
- 无底部 → 可能是简化版，仍属本类型

## 六种子类型

### A. 排行榜型（约40%，最常见）

排名列表 + 底部个人统计，是最典型的形态。

**典型UI**：UI_RankActServerRank、UI_BanquetRank、UI_BeautyCompRank、UI_ActivityRank、UI_SpaceRank、UI_SilkroadRank、UI_GuildHaggleRank

**特征**：表头含"排名/玩家/积分"列，条目纵向排列，底部固定"自己"的数据条。

### B. 任务列表型（约15%）

任务条目列表，每项含进度+操作按钮。

**典型UI**：UI_CoatingTask、UI_MarketWarTask、UI_ActKingOfSnowTask、UI_ActSnowManTask、UI_RelayTrainTask

**特征**：条目标题+进度条（如"1/10"）+ 状态标签（已完成/已领取）。

### C. 奖励列表型（约10%）

奖励条目展示，常带"一键领取"按钮。

**典型UI**：UI_RelayRaceRwds、UI_BanquetPopularityReward、UI_BeautyCompAwardPreview、UI_TopLeagueReward、UI_SilkroadRankReward

### D. 记录/日志型（约10%）

历史记录或战报列表。

**典型UI**：UI_BanquetRecord、UI_VisitRecord、UI_ActApexPvpRecord、UI_TopLeagueRecord、UI_BlockBattleRecord、UI_TradeWarReport

### E. 选择列表型（约10%）

用于选择的条目列表（门客/店铺/成员）。

**典型UI**：UI_StreetFightMarket、UI_BanquetGuest、UI_GuildWarGroup、UI_SilkroadMemberSelect、UI_AuthSelect

### F. 详情/设置型（约15%）

信息展示或设置配置的条目列表。

**典型UI**：UI_MountAureole、UI_AssistantRouteTrade、UI_ResRetrieve、UI_MarketRevenue、UI_MarketSuggest、UI_ExpandFogMap、UI_GachaPreview

## 常见混淆类型及区分

### vs 上标题-中横列表-下信息（最重要的区分！）

| 特征 | 中竖列表 | 中横列表 |
|------|---------|---------|
| 条目方向 | **垂直**（↑↓滚动） | **水平**（←→滚动） |
| 条目形状 | **横向长条** | 竖向卡片 |
| 表头 | 常见（排名/名字/数值） | 罕见 |
| 底栏 | 个人统计/倒计时/一键领取 | 确认/出战/购买按钮 |
| 用途 | 排行榜/记录/数据展示 | 选择/配置/预览 |

**一句话区分**：竖列表看数据（排行榜），横列表做选择（角色/商品）。

### vs T型-上标题-左标签-右竖列表

| 特征 | 中竖列表 | T型-左标签-右竖列表 |
|------|---------|-------------------|
| 左侧标签 | 无 | 有竖向标签导航 |
| 结构 | 单列竖列表 | 左标签 + 右竖列表 |

### vs 确认弹框

| 特征 | 中竖列表 | 确认弹框 |
|------|---------|---------|
| 模态 | 全屏/大面板 | 居中弹窗 |
| 列表 | 有 | 无 |

## 判断口诀

> **上标居中下信息，中间条目竖着立。横向长条上下滚，排行榜类最典型。**

## 主要特征

- 纵向三段式：上标题 + 中竖列表 + 下信息
- 中部条目**垂直堆叠**、横向长条形状、上下滚动
- 排行榜是最典型的子类型（约40%）
- 底栏常为个人统计/倒计时/一键操作
- 与「中横列表」的核心区别：列表方向（竖 vs 横）
