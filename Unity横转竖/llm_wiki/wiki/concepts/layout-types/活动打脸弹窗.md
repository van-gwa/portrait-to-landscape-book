# 活动打脸弹窗

## 定义

"打脸"意为突然弹出、直击用户注意力的**营销/活动类弹窗**。以全屏半透明遮罩+居中弹窗为载体，通过角色立绘、倒计时、促销标签等元素推动用户参与活动或完成付费。属弹框类大类，共 30 个 UI。

**命名含义**：活动 = 营销/事件驱动，打脸 = 强势弹出引起注意，弹窗 = 居中浮窗形态。

## 布局结构

```
┌──────────────────────────────────┐
│    全屏半透明黑色遮罩           │
│  ┌────────────────────────┐     │
│  │ [关闭]         [资源] │     │
│  │                        │     │
│  │  [角色立绘]  活动标题  │     │
│  │              ┌──────┐ │     │
│  │  倒计时      │ 商品  │ │     │
│  │  23:59:59    │ 图标  │ │     │
│  │              └──────┘ │     │
│  │     [折扣标签]        │     │
│  │  [               ]    │     │
│  │  [  ¥68 购买   ]     │     │
│  └────────────────────────┘     │
└──────────────────────────────────┘
```

## 三步判断法（基于30个UI图片分析）

### 第1步：有没有全屏半透明遮罩？

界面背后是否有**全屏黑色半透明遮罩**覆盖原场景？
- 有（100%满足）→ 继续
- 否 → 不是弹窗类型

### 第2步：是不是居中弹窗？

主体内容是否以**居中弹窗**形式浮在遮罩之上？
- 是弹窗（非全屏页面） → 继续
- 是全屏页面 → 考虑全屏类

### 第3步：是活动营销性质还是系统结算？

弹窗的目的是什么？
- **活动推广/商品购买/招募/特权卡** → **活动打脸弹窗** ✓
- **系统结算/战斗结果/升级成功** → 展示-横条-条内信息

关键区分元素（满足2+即本类型）：
- 角色立绘/精美插画
- 倒计时/限时标签
- 折扣标签（5折/限时优惠）
- 购买按钮（¥XX元购买）
- 活动名称/活动说明

## 四种常见子类型

### A. 商品购买型（约35%，最常见）

直接售卖门客/道具/礼包，含角色立绘+折扣+购买按钮。
**典型UI**：UI_GuestBuy、UI_ActivityBuyGuest2、UI_GuestRechargeBuy、UI_SurpriseGift、UI_VWo12

### B. 活动引导型（约25%）

引导用户参与活动，含倒计时+活动说明+前往按钮。
**典型UI**：UI_AddShortCut、UI_ALiFirstVisit、UI_AliRepetitionVisit、UI_SilkCaravan、UI_SilkroadAppraisal、UI_RankActAd、UI_RankActPreview

### C. 奖励领取型（约20%）

展示可领取奖励，含任务列表+已领取/领取按钮。
**典型UI**：UI_ActivityBuyGuest、UI_ALiPCLogIn、UI_BossRedPacketGetReward、UI_MountActive、UI_GuestAureole、UI_ActivityUpRedTip

### D. 特权卡型（约20%）

月卡/特权卡购买，含有效期+权益说明+购买按钮。
**典型UI**：UI_PrivilegeCardInfo、UI_PrivilegeMonthlyCard、UI_AirRoutePrivilege、UI_BattlePassBuyLicense、UI_GuestManagerSkinBuy、UI_RecruitMain

## 常见混淆类型及区分

### vs 展示-横条-条内信息（最重要的区分！）

| 特征 | 活动打脸弹窗 | 展示-横条-条内信息 |
|------|------------|-------------------|
| **目的** | 营销推广/付费转化 | 系统结算/结果通知 |
| **倒计时** | 常见（限时活动） | 罕见 |
| **折扣标签** | 常见（5折/限时） | 无 |
| **购买按钮** | ¥XX元购买 | 确认/再次/关闭 |
| **角色立绘** | 精美大图 | 通常无 |
| **视觉风格** | 华丽装饰、古风卷轴 | 简洁横条面板 |

### vs 确认弹框

| 特征 | 活动打脸弹窗 | 确认弹框 |
|------|------------|---------|
| 复杂度 | 高（多元素） | 低（简单提问） |
| 按钮 | 单按钮（购买/前往） | 双按钮（确认+取消） |
| 装饰 | 丰富 | 简洁 |

### vs 使用弹框

| 特征 | 活动打脸弹窗 | 使用弹框 |
|------|------------|---------|
| 场景 | 活动/商店 | 道具使用 |
| 交互 | 购买/领取 | 选择数量+使用 |

## 主要特征

- 100% 全屏半透明黑色遮罩
- 居中弹窗形式
- 营销驱动的视觉风格（华丽/古风卷轴/装饰丰富）
- 常见：角色立绘、倒计时、折扣标签、¥购买按钮
- "打脸"= 突然弹出、不容忽视
