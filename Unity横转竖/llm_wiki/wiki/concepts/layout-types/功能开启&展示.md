# 功能开启&展示

## 定义

「功能开启&展示」是**新功能/新关卡/新奖励的解锁宣告弹窗**——全屏遮罩 + 居中展示 + 庆祝/宣告风格。属弹框类大类，共14个UI。

**核心判据**："解锁/开启/获得"主题 + 居中宣告式弹窗。

## 布局结构

```
│      全屏黑色遮罩            │

│    │  ✦ 新功能开启！✦ │      │

```

## 三步判断法（5样本+1验证，盲测1/1通过）

### 第1步：主题是不是"解锁/开启/获得"？

- UI目的是宣告新功能开启、新关卡解锁、新奖励获得 → 继续
- 是操作确认/物品使用/活动推广 → 考虑其他弹框类

### 第2步：是不是居中宣告式弹窗？

- 全屏遮罩 + 居中弹窗 + 庆祝/宣告风格
- 与确认弹框的区别：无"确认/取消"双按钮
- 与使用弹框的区别：无数量选择器

### 第3步：确认命名特征

- 常见关键词：Open、Unlock、Get、Upgrade、Succ
- 是 → **功能开启&展示** ✓

## 常见混淆

### vs 确认弹框

| 特征 | 功能开启&展示 | 确认弹框 |
|------|------------|---------|
| 目的 | 宣告/通知 | 确认决策 |
| 按钮 | 单按钮/自动关闭 | 双按钮/单确认 |

### vs 展示-横条-条内信息

| 特征 | 功能开启&展示 | 展示-横条-条内信息 |
|------|------------|-------------------|
| 场景 | 首次解锁/开启 | 每次操作结果 |
| 装饰 | 庆祝/宣告风格 | 艺术字+丝带 |

## 判断口诀

> **遮罩居中宣告弹，新功新关新奖展。解锁开启获得主题，功能开启即此般。**
