# UI 分类一致性检查 - 定时任务处理指令

当你收到来自 openCLaw 的 `UIClassificationVerify` 系统事件时，执行以下流程：

## 流程概述

```
1. /clear                    (清理上下文)
2. 随机选择 UI 图片          (从指定目录)
3. 阅读和分析图片            (判断 UI 类型)
4. 查询已有分类              (在 llm_wiki 中搜索)
5. 对比结果                  (检查一致性)
6. 记录到每日执行日志        (execution_logs/YYYY-MM-DD.md)
7. 记录和同步到汇总页面      (更新 Wiki QA 页面)
```

---

## 详细步骤

### Step 1：初始化

```bash
/clear
```

**目的**：清理上一次的上下文，确保新检查的独立性

### Step 2：选择随机 UI 图片

```

找出所有 UI_*.png 的图片文件
随机选择一张
记录：图片名称、完整路径
```

```
已选择 UI 图片：
```

### Step 3：分析图片内容

**阅读选中的 UI 图片文件**

根据以下维度分析 UI 的布局类型：

#### 1. 检查四边框
- ✅ 有四边框 → 「四边木栏系列」的某种类型
- ❌ 无四边框 → 「其他系列」的某种类型

#### 2. 分析左侧结构
- 标签页/Tab → 左侧导航
- 立绘/形象 → 左侧展示区
- 信息块 → 左侧信息区
- 无左侧 → 标准单侧布局

#### 3. 分析右侧结构
- 宽列表（W > 1000） → 列表区
- 文本内容 → 信息区
- 立绘 → 展示区
- 无右侧 → 单侧布局

#### 4. 识别其他特征
- 标题块 → 顶部标题
- 标签页 → 分类导航
- 抽屉 → 隐藏菜单
- 信息栏 → 底部信息

**输出格式**：
```
【UI 图片分析】

UI 名称：[UI_XXX]
四边框：[有/无] ✓
左侧结构：[描述]
右侧结构：[描述]
其他特征：[列举]

【初步判断】
布局类型：[判断的细分类型，如「四边木栏+左右-宽列表」]
置信度：[高/中/低]
关键依据：[列举 2-3 个最关键的特征]
```

---

### Step 4：查询已有分类

**在 llm_wiki 中搜索该 UI 的现有分类**

```
搜索位置：
  1. llm_wiki/wiki/entities/ 目录
  2. 文件名可能是 [UI_XXX].md
  3. 也可能在其他实体页面中被引用

搜索内容：
  - "UI_XXX" 的字符串
  - "布局类型" 或 "layout type" 字段
  - 关联的布局类型链接
```

**输出格式**：
```
【数据库查询结果】

UI 名称：[UI_XXX]
查询状态：[已找到/未找到]

如果已找到：
  文件：[llm_wiki/wiki/entities/UI_XXX.md]
  已有分类：[记录的布局类型]
  最后更新：[日期]

如果未找到：
  状态：此 UI 尚未分类
  建议：可考虑新建实体页面
```

---

### Step 5：对比结果

**判断分析结果与已有分类的一致性**

```
对比标准：

1. 完全一致
   - 分析结果 == 已有分类
   - 结果：✅ 通过

2. 同大类不同细分
   - 分析：「四边木栏+左中右」
   - 已有：「四边木栏+左右-宽列表」
   - 判断：可接受的差异（都是四边木栏系列）
   - 结果：✅ 通过（记录为小差异）

3. 完全不同
   - 分析：「上标题系列」
   - 已有：「弹框类」
   - 结果：⚠️ 问题，需要处理

4. 一个未分类
   - 分析：「四边木栏+左右-宽列表」
   - 已有：未分类
   - 结果：⚠️ 待确认（可能需要新建页面）
```

**输出格式**：
```
【对比结果】

分析结果：[XXX]
已有分类：[XXX]
对比结论：[一致/小差异/不一致/未分类]

一致性评分：[✅/⚠️/❌]
  - ✅ 完全一致或同大类
  - ⚠️ 小差异或存在疑问
  - ❌ 完全矛盾
```

### Step 6：输出结果并自动记录日志

**输出标准化结果，然后执行日志记录脚本**

#### 6.1 输出结构化数据

输出以下格式（供脚本自动处理）：

```
[EXECUTION_LOG_MARKER]
TIME: HH:mm:ss
UI_NAME: [UI_XXX]
INITIAL_TYPE: [类型]
ACTUAL_TYPE: [类型]
RESULT: [✅/⚠️/❌]
NOTES: [备注]
[END_EXECUTION_LOG_MARKER]
```

**示例：**

```
[EXECUTION_LOG_MARKER]
TIME: 10:11:45
UI_NAME: UI_GachaLimitWish
INITIAL_TYPE: 展示-横条-左图-右信息
ACTUAL_TYPE: T型-上标题-左-右
RESULT: ⚠️
NOTES: 维度差异，均无误
[END_EXECUTION_LOG_MARKER]
```

#### 6.2 执行日志记录脚本

在输出标记后，执行以下 PowerShell 命令来自动更新日志：

```powershell
powershell -File "C:\Users\guowe\.openclaw\scripts\ui-classification-auto-log.ps1" `
  -UIName "[UI_XXX]" `
  -InitialType "[初判类型]" `
  -ActualType "[实际类型]" `
  -Result "[✅/⚠️/❌]" `
  -Notes "[备注]"
```

执行结果会打印确认信息。

#### 6.3 继续输出可读的总结

在脚本执行后，可以继续输出人类可读的检查总结。

## 注意事项

1. **图片路径错误**
   - 如果目录不存在或为空，记录错误并报告
   - 不要强行创建目录

2. **分类不确定**
   - 如果置信度低于「中」，标记为「需要人工验证」
   - 不要强行判断

3. **数据库查询失败**
   - 如果搜索失败，标记为「查询异常」
   - 继续进行其他步骤

4. **每次独立**
   - 每次运行都从 /clear 开始
   - 不依赖上一次的结果
   - 独立记录每次检查

## 定时触发方式

openCLaw 每 3 分钟会发送系统事件：
```json
{
  "kind": "systemEvent",
  "text": "UIClassificationVerify"
}
```

当你收到此事件时，按照上述流程执行。
