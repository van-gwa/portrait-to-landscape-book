# 战斗

## 定义

「战斗」是游戏中唯一的**全屏斜45度等距视角战斗界面**，玩家与敌方呈对角线对峙。属特殊玩法大类，共 9 个 UI，布局高度统一。

**核心识别特征**：等距视角 + 左下玩家 vs 右上敌方的对角线对峙 + 四角UI元素环绕。

## 布局结构

```
┌──────────────────────────────────┐
│ [返回]               [敌血条]   │  ← 顶部：返回+敌方血量
│ [玩家]                          │
│                                  │
│     ╲  斜45度等距场景  ╱        │
│       ╲              ╱          │  ← 中间：对角线战场
│   玩家  ╲          ╱  敌方      │
│   (左下)  ╲      ╱  (右上)     │
│            ╲  ╱                │
│             ╱ ╲                │
│                                  │
│ [士气条]    [技能按钮]  [跳过]  │  ← 底部：士气+技能+跳过
└──────────────────────────────────┘
```

## 三步判断法（基于9个UI全量分析）

### 第1步：是不是全屏界面？

战斗UI**100%是全屏**（非弹窗/浮窗），背景为完整的等距视角场景。

- 是全屏 → 继续
- 是弹窗/遮罩 → 不是战斗类型

### 第2步：是不是斜45度等距对角线对峙？

场景采用**等距视角**（Isometric），双方呈**左下vs右上对角线**分布：
- 玩家方：左下区域
- 敌方：右上区域

9个UI全部满足此特征。

### 第3步：四角UI布局是否一致？

检查四个角落的固定元素：

| 位置 | 元素 |
|------|------|
| **左上** | 返回按钮(云朵造型) + 玩家简要信息 |
| **右上** | 敌方血条(红色长条) + 敌方名称/等级 |
| **左下** | 绿色士气条 + 聊天/商会按钮 |
| **右下** | 橙色"跳过"按钮 + 回合状态提示 |

四角齐全 → **战斗** ✓

## 常见混淆类型及区分

### vs 其他全屏类型

| 特征 | 战斗 | 全屏背景界面 | 登录加载全屏 |
|------|------|------------|------------|
| 视角 | 斜45度等距 | 平面/正面 | 静态图 |
| 对角线对峙 | 有 | 无 | 无 |
| 四角UI固定 | 有（返回/血条/士气/跳过） | 不固定 | 无 |
| 战斗元素 | 血条+士气+技能+跳过 | 无 | 无 |

**一句话**：看到左下绿色士气条 + 右下橙色跳过按钮 + 斜45度场景 = 战斗类型。

## 判断口诀

> **全屏斜角对角线，左下玩家右上敌。士气绿条左下立，橙色跳过右下栖。**

## 主要特征

- 全屏（非弹窗），9个UI全部一致
- 等距视角（Isometric）场景
- 左下vs右上对角线对峙
- 四个角落UI元素高度固定
- 战斗专属元素：血条、士气条、跳过按钮、技能按钮
