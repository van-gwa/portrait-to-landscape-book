# UI 分类系统 - 启动检查清单

## 📋 必需操作（按顺序执行）

### ✅ 步骤 1：配置 API 密钥

- [ ] 打开 PowerShell（**管理员权限**）
- [ ] 复制并执行以下命令：
  ```powershell
  powershell -ExecutionPolicy Bypass -File "C:\Users\guowe\.openclaw\scripts\setup-api-from-cursor-permanent.ps1"
  ```
- [ ] 脚本会自动从 Cursor 中提取 API 密钥
- [ ] 脚本完成后，显示"✅ 配置完成！"

**预期输出：**
- "✓ API 密钥来源" → 成功找到密钥
- "✅ 环境变量已设置为用户级别"
- 日志文件位置提示

---

### ✅ 步骤 2：重启 PowerShell

- [ ] 关闭当前 PowerShell 窗口
- [ ] 打开新的 PowerShell 窗口
- [ ] 验证环境变量已设置：
  ```powershell
  echo $env:CLAUDE_API_KEY
  ```
- [ ] 应该看到：`sk-ant-xxxxx...`

**如果为空：**
- 重新执行步骤1
- 确保使用了管理员权限

---

### ✅ 步骤 3：等待自动执行

- [ ] Task Scheduler 会在下一个 3 分钟周期运行任务
- [ ] 不需要任何额外操作

## ❌ 问题排除

### 问题：脚本说"CLAUDE_API_KEY 环境变量未设置"

**解决方案：**
1. 确保已完成步骤 1（配置脚本）
2. 确保使用了管理员权限运行
3. 重启 PowerShell（步骤 2）
4. 验证：`echo $env:CLAUDE_API_KEY`

### 问题：配置脚本找不到 Cursor 设置

**检查项：**
1. Cursor 是否已安装？
2. Cursor 中是否已配置 Claude API 密钥？
3. 设置路径：`%APPDATA%\Code\User\settings.json`

**如果配置脚本失败，可以手动输入密钥：**
- 脚本会提示"输入 API Key (sk-ant-xxx)"
- 从 Cursor 中复制密钥并粘贴

### 问题：Task Scheduler 任务没有运行

**检查任务状态：**
```powershell
Get-ScheduledTask -TaskName "UIClassificationCheck" | Select-Object State, Enabled
Get-ScheduledTaskInfo -TaskName "UIClassificationCheck"
```

**手动触发任务测试：**
```powershell
Start-ScheduledTask -TaskName "UIClassificationCheck"
```

**然后检查日志：**
```powershell
Get-Content "C:\Users\guowe\.openclaw\logs\ui-classification-results.log" -Tail 10
```
