# UI 布局类型分类指南

**目的**: 快速识别任何新UI属于哪种布局类型

**核心分类维度**: 
1. **主体结构** - 弹框/全屏/窗口/面板等
2. **布局方向** - 左右/上下/左-中-右/T型等
3. **装饰元素** - 木栏/艺术边框/背景装饰等
4. **内容类型** - 列表/网格/单项/信息展示等

---

## 第一层分类：主体结构

### 1️⃣ **弹框类** (4种 | 141 UIs)
弹框是独立的浮窗，通常居中，用于临时交互或确认

| 布局类型 | UIs | 特征 |
|--------|-----|------|
| 确认弹框 | 87 | 单行/多行文本 + 底部按钮 |
| 使用弹框 | 24 | 配置说明 + 使用功能 + 按钮 |
| 活动打脸弹窗 | 30 | 活动奖励展示 + 动画效果 |

**识别方式**: 
- 窗口相对较小
- 通常有明确的标题
- 底部有操作按钮（确认/取消）
- 背景通常有半透明遮罩

## 使用流程图

```
📸 看到一个新UI
       ↓
   [第一步] 判断主体结构
   ├─ 小的浮窗？ → 弹框类
   ├─ 占满整屏？ → 全屏类
   └─ 中等窗口？ → 面板类
       ↓
   [第二步] 分析内容布局方向
   ├─ 左边列表/标签 + 右边内容？ → 左右布局
   ├─ 上面标题 + 下面列表？ → 上下布局
   ├─ 上标题 + 下左右分割？ → T型布局
   └─ 其他复杂组合？ → 复杂布局
       ↓
   [第三步] 检查是否有装饰元素
   ├─ 黄棕色木栏边框？ → 木栏类
   ├─ 左右彩色装饰？ → 艺术设计类
   ├─ 灯笼/屏风/墙？ → 特殊装饰
   └─ 无特殊装饰？ → 完成分类
       ↓
   [第四步] 精确细分
   └─ 根据内部列表方向/标签位置等细节确定
```

## 分类命名规律

### 命名模式
```
[装饰元素] + [主体方向] + [内容类型] + [位置/细节]

示例：
- "四边木栏（蒸笼标签）+左中右"
  装饰: 四边木栏(蒸笼标签)
  方向: 左中右(3列)
  
- "上标题-中竖列表-下信息"
  方向: 上-中-下(3行)
  内容: 标题/竖列表/信息
  
- "展示-横条-左图-右信息"
  类型: 展示
  结构: 横条
  内容: 左图右信息
```

### 常用词汇
- **左/右/上/下** - 方向
- **中** - 中间区域
- **竖列表/横列表** - 列表方向（竖=纵向，横=横向）
- **标签** - 分类标签/标题栏
- **标题** - 顶部标题
- **信息** - 底部信息区
- **框** - 边框/容器
- **展示** - 显示/呈现类
- **条** - 条形/条纹元素

**容易犯的错误**：
```

错误结论 → 左标签页-右框内-上标题-中竖列表-下信息 ✗
```

**正确的判断过程**：
```
看图 → 先看上方

↓

↓

↓
第二步: 上面是标题，下面是左标签+右竖列表
↓
最终: T型-上标题-左标签-右竖列表 ✓

## 实际应用建议

### 当你看到一个新的UI图片时：

1. **先看整体** → 弹框/全屏/窗口？
2. **再看内容布局** → 左右/上下/T型/其他？
3. **查看装饰** → 有木栏/艺术边框/特殊装饰吗？
4. **对标同类** → 打开本文档中对应类型的示例对比
5. **细节确认** → 内部列表方向、标签位置等

### 如果不确定：
- 查看 [[UI_Layout_Classification_Index]] 中的相关类型笔记
- 打开笔记中的示例 UI 截图对比
- 关注布局的"最主要特征"而不是细枝末节
