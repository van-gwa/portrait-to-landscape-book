# UI 布局类型快速判断流程

## 🚀 快速判断三步法

```
第一眼看大小和位置
   ↓
确定主体结构
   ↓
分析内容方向
   ↓
检查装饰元素
   ↓
精确匹配类型
```

---

## 判断流程表 v1.0

### Step 1: 窗口大小和位置

| 特征 | 分类 | 示例类型 |
|-----|------|--------|
| **小浮窗，中心位置** | 弹框类 | 确认弹框、使用弹框、活动打脸弹窗 |
| **占满整个屏幕** | 全屏类 | 全屏背景界面、全屏入口、登录加载全屏 |
| **中等大小窗口** | 面板/窗口类 | panel、一体框、panel+入口 |
| **提示类小窗口** | 特殊类 | tips、红框方形玩法入口 |

---

### Step 2: 判断主结构方向

#### 👉 如果是**弹框**（小浮窗）

| 内容结构 | 布局类型 |
|--------|--------|
| 标题 + 提示文字 + 确认/取消按钮 | **确认弹框** |
| 标题 + 使用说明 + 功能配置 + 按钮 | **使用弹框** |
| 标题 + 奖励展示 + 特效 | **活动打脸弹窗** |

---

#### 👉 如果是**全屏**（占满屏幕）

| 内容结构 | 布局类型 |
|--------|--------|
| 全屏背景 + 规则内容区 | **全屏背景界面** |
| 全屏 + 单个大按钮 | **全屏入口** |
| 全屏背景 + 奖励展示动画 | **全屏获得新奖励** |
| 灯笼+彩带装饰 | **灯笼标签+彩带+全屏** |
| 登录/加载画面 | **登录加载全屏** |

---

#### 👉 如果是**中等窗口**

首先判断**主体布局方向**：

```
┌─────────────────────────────────────────┐
│  是否左侧有竖列表/标签？                    │
├─────────────────────────────────────────┤
│  YES → 左右布局                           │
│  NO  → 继续判断                           │
└─────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────┐
│  是否上方有标题？                          │
├─────────────────────────────────────────┤
│  YES → 上下布局（或T型）                   │
│  NO  → 其他布局                           │
└─────────────────────────────────────────┘
```

---

### Step 3: 根据方向精确分类

#### ⬅️➡️ 左右布局系列

```
看左边内容
├─ 竖列表？
│  ├─ 左竖列表 + 右信息
│  │  ├─ 有标题在左上？ → 左标签+右竖列表
│  │  └─ 无标题？ → 左标签-右信息
│  │
│  └─ 左竖列表 + 中大区域 + 右信息
│     ├─ 有木栏边框？ → 四边木栏（蒸笼标签）+左中右 ⭐
│     └─ 无木栏？ → 左-中-右
│
├─ 大图？
│  ├─ 图很大 + 右竖列表 → 左广告图-右列表
│  └─ 左图 + 右竖列表 → 左图-右竖列表
│
├─ 装饰框？
│  ├─ 左边框装饰 → 左艺术设计+左-右
│  └─ 左右都有装饰 → 左右艺术设计+一体
│
└─ 两个对比项？ → 左右对比
```

---

#### ⬆️⬇️ 上下布局系列

```
看上方内容
├─ 标题？
│  ├─ YES:
│  │  └─ 下方是什么？
│  │     ├─ 竖列表？ → 上标题-中竖列表-下信息 ⭐⭐⭐ (92个)
│  │     │
│  │     ├─ 横列表？ → 上标题-中横列表-下信息 ⭐⭐ (57个)
│  │     │
│  │     ├─ 有标签在中上？
│  │     │  └─ YES → 上标题-中上标签-中竖列表-下信息
│  │     │
│  │     └─ 无更多结构？ → 上标题+下横列表
│  │
│  └─ NO: 继续判断
│
└─ 多个横条形的项？
   ├─ 条内直接放文字 → 展示-横条-条内信息 ⭐⭐ (63个)
   ├─ 条内左图右文 → 展示-横条-左图-右信息
   ├─ 条内上图下文 → 展示-横条-上图-下信息
   └─ 条内左右对比 → 展示-横条-左右对比
```

---

#### 🎯 T型布局系列

```
T型布局特征：
上 = 标题行
下 = 左右分割

├─ 下方是左/右两列？
│  └─ YES:
│     ├─ 右边是竖列表？ → T型-上标题-左标签-右竖列表 (28个)
│     └─ 下左右都是内容 → T型-上标题-左-右 (26个)
└─ NO: 不是T型
```

### Step 4: 检查装饰元素

#### 🪵 木栏装饰

```
看边框
├─ 四周有黄/棕色木栏框？
│  └─ YES:
│     └─ 看内部结构:
│        ├─ 上方有多个标签页（蒸笼状）？
│        │  └─ YES → 有"蒸笼标签"标记
│        │
│        ├─ 内部是左中右？ → 四边木栏（蒸笼标签）+左中右
│        ├─ 内部是横列表？ → 四边木栏（蒸笼标签）+横列表
│        ├─ 内部是网格？ → 四边木栏（蒸笼标签）+格子排
│        ├─ 木栏在底部？ → 四边木栏（蒸笼标签）底
│        ├─ 有功能入口？ → 四边木栏（蒸笼标签）+入口
│        └─ 特殊的三区域？ → 四边木栏（蒸笼标签）+左标签+中图+右信息
│
└─ NO: 继续检查其他装饰
```

#### 🎨 艺术设计装饰

```
看左右两边
├─ 左右边有装饰图案框？
│  └─ YES:
│     └─ 中间是什么？
│        ├─ 横列表 → 左右艺术设计+横列表
│        ├─ 竖列表 → 左右艺术设计+竖列表
│        └─ 一体内容 → 左右艺术设计+一体
│
└─ 只有左边装饰？ → 左艺术设计+左-右
```

#### 🏮 其他特殊装饰

```
├─ 灯笼图案 + 彩带？ → 灯笼标签+彩带+全屏
├─ 屏风纹理背景？ → 木屏风+横列表
├─ 墙体纹理 + 小旗？ → 墙+小旗+横列表
└─ 大背景 + 上层展示？ → 大背景-展示
```

## 常见判断陷阱 ⚠️

### 陷阱 1: 混淆"标题"的含义
```
❌ "有文字在上面就是上标题"
✅ "标题是单独的标题栏，区别于列表内容"
```

### 陷阱 2: 忽视装饰元素
```
❌ "左右列表就是左-右布局"
✅ "左右列表+木栏框 = 四边木栏（蒸笼标签）+左中右"
```

### 陷阱 3: 混淆列表方向
```
❌ "有好几行就是竖列表"
✅ 竖列表 = 纵向滚动，项目一个接一个上下排列
✅ 横列表 = 横向排列，项目左右排列或在同一行
```

### 陷阱 4: 弹框 vs 小窗口
```
❌ "小的窗口就是弹框"
✅ 弹框 = 浮在最上层，有遮罩，通常有按钮
✅ 小窗口 = 独立的面板，不一定有遮罩
```

### 陷阱 5: 忽视多标签
```
❌ "上面有标签就是上标题"
✅ 多个标签分页 = "蒸笼标签"标记
✅ 单个标题 = 普通"上标题"
```

### 🚨 陷阱 6: 混淆T型 vs 左标签页（重要！）
```
❌ "左侧有标签列表 + 右侧有内容" = "左标签页-右框内"
✅ 区别方法：看上方的标题

T型-上标题-左标签-右竖列表：
┌─────────────────────────┐
│    标题（贯穿全宽）       │  ← 关键：标题线贯穿整个宽度，是全局分割线
├─────┬───────────────────┤
│左标签 │    右竖列表       │
│列表 │                   │
└─────┴───────────────────┘

左标签页-右框内-上标题-中竖列表-下信息：
┌────┬──────────────────┐
│    │    标题（只在框内）│  ← 错误：标题只在右框内，不是全局分割
│左   │  ├──────────────┤
│标签 │  │竖列表       │
│页   │  ├──────────────┤
│    │  │下信息       │
│    │  └──────────────┘
└────┴──────────────────┘

【实际案例】UI_BeautyTravel:
- ✅ 上面有"旅游与颜"标题 → 贯穿全宽 → T型的"上"
- ✅ 下面左是标签列表（美妆资料/美女名人等）
- ✅ 下面右是竖列表区域
- 结论：T型-上标题-左标签-右竖列表 ✓
```

## 当你还是不确定时...

### 用排除法
1. 不是弹框吗？ → 检查是否全屏/面板
2. 不是全屏吗？ → 肯定是面板/窗口
3. 是面板吗？ → 看左右/上下/T型哪个最接近
4. 有装饰元素吗？ → 加上装饰限定

### 对标现有类型
1. 打开 [[UI_Layout_Classification_Index]]
2. 查看相似的布局类型
3. 打开那个类型的笔记
4. 看该类型包含的实际 UI 截图
5. 跟你的 UI 对比

### 使用数量法则
- 新UI最可能属于数量多的类型
- 前5大类型就覆盖了 ~50% 的UI
- 检查这5个类型通常能确定答案

## 实战练习

### 练习 1: UI_Bag
**提示**: 左黄框+中大区域+右信息栏
**流程**: 中等窗口 → 左右结构 → 有木栏框 → **四边木栏（蒸笼标签）+左中右** ✓

### 练习 2: UI_Achievement  
**提示**: 多条形，每条上面标签名，内部进度条
**流程**: 中等窗口 → 上下结构 → 横条形 → 条内信息 → **展示-横条-条内信息** ✓

### 练习 3: UI_BagComposeTips
**提示**: 中间浮窗，标题"合成提示"，下面确认/取消
**流程**: 小浮窗 → 弹框 → 标题+文字+按钮 → **确认弹框** ✓

### 练习 4: UI_RankActServerRank
**提示**: 标题"医服排行" → 竖列表 → 底部按钮
**流程**: 中等窗口 → 上下结构 → 标题+竖列表+底部 → **上标题-中竖列表-下信息** ✓
