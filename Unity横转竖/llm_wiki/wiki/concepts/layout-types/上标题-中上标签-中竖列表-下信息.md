# 上标题-中上标签-中竖列表-下信息

## 定义

「上标题-中上标签-中竖列表-下信息」是上标题系列中**最复杂的四段式**——标题 + 次级标签行 + 竖列表 + 底部信息。与「上标题-中竖列表-下信息」的区别：多了一层**横向标签行**。共11个UI。

## 布局结构

```
┌──────────────────────────────┐
│         标题                 │  ← 第一层：上标题
├──────────────────────────────┤
│ [标签A] [标签B] [标签C]     │  ← 第二层：中上标签行
├──────────────────────────────┤
│ ┌──────────────────────────┐ │
│ │ 条目1                   │ │
│ │ 条目2                   │ │  ← 第三层：中竖列表
│ │ ...                     │ │
│ └──────────────────────────┘ │
├──────────────────────────────┤
│   底部信息/统计              │  ← 第四层：下信息
└──────────────────────────────┘
```

## 三步判断法（4样本+1验证，通过）

### 第1步：顶部有没有标题？

- 有顶部标题 → 继续

### 第2步：标题下面有没有横向标签行？

- 标题下方有**水平排列的标签/Tab按钮**（不是左侧竖向标签）→ 继续
- 无标签行 → 考虑上标题-中竖列表-下信息

### 第3步：标签行下面是竖列表吗？

- 竖列表 + 底部信息/统计 → **上标题-中上标签-中竖列表-下信息** ✓

---

## 判断口诀

> **上标下标签横排，再下竖列滚滚来。四段结构最复杂，上标系列高配在。**
