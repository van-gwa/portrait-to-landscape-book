# UI 布局快速判断流程

**目的**：3-5分钟内快速判断任何UI属于哪个布局类型

**适用**：日常工作中快速分类UI

**精准度**：95%+（大多数UI能直接判断）

---

## 🚀 超快速三步法（30秒）

### 看一眼 → 比一比 → 得答案

```
第一眼
  ↓
看大小和位置
  ├─ 小浮窗？→ 弹框类
  ├─ 全屏？ → 全屏类
  └─ 中等？ → 继续
  ↓
看内容布局
  ├─ 左右分？ → 左右布局
  ├─ 上中下？ → 上下布局
  ├─ 上横线+下左右？ → T型布局
  └─ 其他？ → 打开速查表对比
  ↓
看有没有装饰
  ├─ 黄棕木栏？ → 木栏系列
  ├─ 左右彩色？ → 艺术系列
  └─ 没有？ → 确认类型
  ↓
✅ 答案！
```

**大多数UI只需3-5秒就能判断！**

## 📋 标准五步判断流程

### 第1步：确定主体结构（5秒）

看窗口**大小和位置**：

```
🔍 关键问题：这个窗口有多大？

小浮窗（300×400以下）
↓
┌──────────────────┐
│ 标题             │
│ 内容             │ ← 小浮窗，居中显示
│ [按钮1] [按钮2]  │
└──────────────────┘
背景有半透明遮罩

✅ 答案：弹框类（确认弹框/使用弹框/活动弹窗）


占满整个屏幕
↓
┌──────────────────────────┐
│                          │
│  精美背景                │ ← 占满屏幕
│  + 主要内容              │
│                          │
└──────────────────────────┘

✅ 答案：全屏类（全屏背景/全屏入口/等）


中等大小窗口
↓
┌──────────────────┐
│ 标题栏           │
│ ┌──────────────┤
│ │ 内容         │ ← 中等窗口，独立
│ │              │
│ └──────────────┘
└──────────────────┘

✅ 答案：继续第2步，看内容如何布局
```

---

### 第2步：分析布局方向（10秒）

**只对"中等窗口"做这一步。弹框和全屏已经确定了。**

看内容**如何分割**：

```
🔍 关键问题：内容怎么分割？


【情况1】左侧有竖列表/标签 + 右侧有内容
  
┌──────┬──────────────┐
│左    │    右        │
│列表  │    内容      │
│      │              │
└──────┴──────────────┘

✅ 答案：左右布局
进入第3步判断具体类型


【情况2】上标题 + 中内容 + 下操作
  
┌──────────────────────┐
│ 标题（上）           │
├──────────────────────┤
│ 竖列表或横列表（中） │
│                      │
├──────────────────────┤
│ 操作按钮（下）      │
└──────────────────────┘

✅ 答案：上下布局
进入第3步判断具体类型


【情况3】上面一条横线 + 下面左右分割
  
┌──────────────────────┐
│  标题贯穿全宽（上）   │ ← 【关键】贯穿全宽！
├──────┬───────────────┤
│左    │     右        │
│标签  │    列表       │
└──────┴───────────────┘

✅ 答案：T型布局
进入第3步判断具体类型


【情况4】其他复杂组合
  
┌────┬──────────────────┐
│ L  │ ┌──────────────┤
│ a  │ │ 框内         │
│ b  │ │ 有上中下      │
│ e  │ └──────────────┘
│    │
└────┴──────────────────┘

或者：多列网格，或其他特殊布局

✅ 答案：特殊布局
需要更多分析或查表
```

---

### 第3步：检查装饰元素（5秒）

看是否有**特殊的框或装饰**：

```
🔍 关键问题：有没有特殊装饰框？


【情况1】四周有黄棕色的木栏框

╔═════════════════════╗
║ [标签] [标签] [标签] ║ ← 蒸笼标签（多个顶部标签）
╠═════════════════════╣
║ 内容                ║
╚═════════════════════╝
  ↑ 古风木栏

✅ 答案：木栏装饰系列（4边木栏+...）


【情况2】左右两边有彩色装饰框

┌────┐              ┌────┐
│ 🎨 │   中间内容   │ 🎨 │
│    │              │    │
└────┘              └────┘

✅ 答案：艺术设计系列


【情况3】有特殊的灯笼、屏风、墙等装饰

🏮 灯笼标签+彩带
🎋 木屏风+...
🧱 墙+小旗+...

✅ 答案：特殊装饰系列


【情况4】没有特殊装饰，就是纯布局

✅ 答案：无装饰，按布局类型确定
```

---

### 第4步：看细节特征（10秒）

根据前面的分析，进一步确认：

```
🔍 关键问题：具体的排列和元素是什么？


对于上下布局，区分：
├─ 竖列表 (上下排列) → 上标题-中竖列表-下信息
└─ 横列表 (左右排列) → 上标题-中横列表-下信息
                  或 展示-横条-... 系列


对于左右布局，区分：
├─ 左标签+右竖列表 → 左标签+右竖列表
├─ 左大图+右列表 → 左广告图-右列表
├─ 左装饰框+左右 → 左艺术设计+左-右
└─ ...


对于T型布局，区分：
├─ 简单T字 → T型-上标题-左-右
└─ 左标签+右竖列 → T型-上标题-左标签-右竖列表


对于木栏系列，区分：
├─ 左中右结构 → 四边木栏+左中右
├─ 横列表 → 四边木栏+横列表
└─ 网格排列 → 四边木栏+格子排
```

---

### 第5步：对标现有类型（10秒）

打开[[UI_Layout_Quick_Reference]]的速查表，找到最匹配的那个：

```
查表快速定位：

1. 知道大类型吗？
   ├─ 知道 → 在对应分组里找
   └─ 不知道 → 看下面

2. 知道名字的关键词吗？
   ├─ "竖列表" → 在名称里搜"竖列表"
   ├─ "标签页" → 搜"标签页"
   ├─ "木栏" → 搜"木栏"
   └─ ...

3. 如果还是不确定，按复杂度找：
   ├─ 简单类型（tips/panel等）
   ├─ 中等（上标题+竖列表等）
   └─ 复杂（木栏系列等）

4. 看一眼示例UI图片对比
   → 找到最接近的那个！
```

## ⚡ 常见快速路径

### 路径1：我看到了"上中下"结构 (~10秒)

```
         ↓
看中间是什么
   ├─ 竖列表？→ 上标题-中竖列表-下信息 ✓
   ├─ 横列表？→ 上标题-中横列表-下信息 ✓
   ├─ 多个横条？→ 展示-横条-... 系列
   └─ 其他？→ 查表
```

### 路径2：我看到了"左右"结构 (~10秒)

```
         ↓
看有没有装饰
   ├─ 有木栏？→ 四边木栏系列
   ├─ 有彩色装饰？→ 艺术设计系列
   └─ 没有？→ 继续看细节
         ↓
看左边是什么，右边是什么
   ├─ 左标签+右竖列？→ 左标签+右竖列表 ✓
   ├─ 左大图+右列？→ 左广告图-右列表 ✓
   ├─ 左竖+中+右竖？→ 左-中-右 ✓
   └─ ...
```

### 路径3：我看到了"T字"结构 (~10秒)

```
         ↓
【关键】上面的标题贯穿全宽吗？
   ├─ 是 → T型！
   └─ 不是 → 不是T型，重新判断
         ↓
看下面是什么
   ├─ 左标签+右竖列？→ T型-上标题-左标签-右竖列表 ✓
   ├─ 左右都是内容？→ T型-上标题-左-右 ✓
   └─ ...
```

### 路径4：我看到了"小浮窗" (~5秒)

```
         ↓
看内容
   ├─ 标题+文字+按钮？→ 确认弹框 ✓
   ├─ 说明+配置+按钮？→ 使用弹框 ✓
   ├─ 奖励展示+动画？→ 活动打脸弹窗 ✓
   └─ ...
```

## 🚨 容易踩的坑

### 🚨 坑1：T型 vs 左标签页

**关键区别**：看上面的标题范围

```
❌ 错误判断：有左侧标签+右侧内容 = 左标签页
✅ 正确判断：看上面标题是否贯穿全宽

【T型-上标题-左标签-右竖列表】
┌──────────────────────┐
│ 标题（贯穿全宽！）   │ ← 这条线贯穿左右
├──────┬───────────────┤  这是T的关键特征
│左标签│    右竖列表   │
└──────┴───────────────┘


【左标签页-右框内-...】
┌──────┬──────────────┐
│      │ 标题(框内)   │ ← 标题只在框内
│      │ ┌──────────┤  不贯穿全宽
│标签  │ │ 内容     │
│页    │ └──────────┘
└──────┴──────────────┘

实际案例：UI_BeautyTravel
看"旅游与颜"
  ├─ 贯穿全宽？YES
  └─ → T型-上标题-左标签-右竖列表 ✓
```

### 🚨 坑2：忽视装饰元素

```
❌ "我看到左右列表，就判断是'左-右'"
✅ 先检查有没有装饰框！

如果有木栏框 → 四边木栏系列（不是左-右）
如果有彩色装饰 → 艺术设计系列（不是左-右）
如果都没有 → 才是左-右
```

### 🚨 坑3：竖列表 vs 横列表混淆

```
❌ "有好几行就是竖列表"
✅ 看排列方向！

竖列表 = 项目上下排列，纵向滚动
┌──────────────┐
│ ◆ 项目1      │ ← 竖向排列
│ ◆ 项目2      │
│ ◆ 项目3      │
└──────────────┘

横列表 = 项目左右排列，横向滚动
┌──────────────────────────┐
│ ◆项目1  ◆项目2  ◆项目3  │ ← 横向排列
└──────────────────────────┘
```

### 🚨 坑4：弹框 vs 小窗口

```
❌ "小窗口就是弹框"
✅ 弹框有特征！

弹框的特征：
✓ 浮在最上层（z-index高）
✓ 有半透明遮罩
✓ 通常有确认/取消按钮
✓ 居中显示
✓ 不能最小化

小窗口可能没有这些特征
```

### 🚨 坑5：蒸笼标签 vs 普通标签

```
❌ "上面有标签就是蒸笼标签"
✅ 看标签的个数和样式

普通标签：
┌──────────────────┐
│ 标题              │ ← 单一的标题或标签，不可切换
│ 内容              │
└──────────────────┘

蒸笼标签：
┌──────────────────┐
│ [标签1][标签2]   │ ← 多个标签，可切换
│ [标签3][标签4]   │  （像蒸笼一样层层叠叠）
├──────────────────┤
│ 内容              │
└──────────────────┘
```
