# 上标题-中横列表-下信息

## 定义

该布局通过**上标题-中横列表-下信息**的三段式纵向空间组织，实现条目的横向浏览与选择。属上标题系列大类，共 57 个 UI，是排名第5的高频类型。

**命名含义**：
- **上标题**：顶部为标题栏（居中标题+返回+资源）
- **中横列表**：中部为横向排列的条目卡片（可左右滚动）
- **下信息**：底部为补充信息或操作按钮

## 布局结构

```
┌──────────────────────────────┐
│ [返回]    标题文字    [资源] │  ← 上：标题栏（居中标题）
├──────────────────────────────┤
│ ┌────┐ ┌────┐ ┌────┐ ┌────┐ │
│ │卡片│ │卡片│ │卡片│ │卡片→│ │  ← 中：横列表（左右滚动）
│ │ 1  │ │ 2  │ │ 3  │ │ 4  │ │
│ └────┘ └────┘ └────┘ └────┘ │
├──────────────────────────────┤
│   信息说明 / [操作按钮]      │  ← 下：信息栏/按钮栏
└──────────────────────────────┘
```

## 三步判断法（基于57个UI图片分析）

### 第1步：看顶部有没有标题栏？

顶部区域是否包含**居中标题文字**？
- 有（居中标题，左侧常带返回按钮，右侧为资源/货币）→ 继续
- 顶部分布到两侧占据空间 → 可能是其他类型

标题通常不是贯穿全宽的硬质横条，而是**居中文字 + 左右功能元素**构成一条顶部区域。

### 第2步：看中部是不是横向列表？

中部条目是否**水平排列**（可左右滚动）？
- 条目是**横排**的卡片，水平方向展开
- 通常只露出部分条目，暗示可滑动查看更多
- 每个条目是**竖向卡片**（例如：角色卡含头像+名字、商品卡含图标+价格）

**关键区分**：横列表 = 左右滚动，竖列表 = 上下滚动

### 第3步：看底部有没有信息或按钮栏？

底部是否有独立的信息条或操作按钮？
- 常见形式：
  - 选中条目后显示的详情信息（消耗/属性/效果）
  - 功能按钮（确认/出战/购买/前往挑战）
  - 筛选条件/排序选项
- 少数仅留白或装饰背景

三者齐备 → **上标题-中横列表-下信息** ✓

## 七种子类型

根据57个UI的功能语义，分为7种子类型：

### 类型A：角色/门客选择型（约25%，最多）

用于选择角色或门客上阵/出战。
**典型UI**：UI_GuildDunLineUp、UI_TradeWarGuests、UI_SilkroadFightSelectGuest、UI_ExpandDispatch、UI_GuestUp
**特征**：卡片=角色头像+名字+属性，底部="出战"/"委任"按钮

### 类型B：商店/商品型（约15%）

用于展示和购买商品。
**典型UI**：UI_ActivityShop、UI_TopLeagueShop、UI_ActApexPvpShop、UI_ShopCommonActShop
**特征**：卡片=商品图标+价格+限购，底部=价格/积分信息

### 类型C：材料选择型（约15%）

用于选择升级/合成材料。
**典型UI**：UI_CoatingSelectUpLevelMat、UI_CoatingSelectUpStarMat、UI_MountSelectUpStarMat
**特征**：卡片=材料图标+数量+品质，底部="确认选择"按钮+已选数量

### 类型D：预览展示型（约15%）

用于预览奖励/升级结果。
**典型UI**：UI_ActSnowWheelPreview、UI_ActSnowManPreview、UI_MountUpgradeResult、UI_LimitActivityPreview、UI_BattlePassBuyExp
**特征**：卡片=奖励/结果条目，底部=消耗/购买信息

### 类型E：对手/目标选择型（约10%）

用于选择战斗对手或任务目标。
**典型UI**：UI_ActArenaTarget、UI_StreetFightTarget、UI_BlockBattleTarget、UI_StreetFightSolo、UI_ActArenaSolo
**特征**：卡片=对手店铺+实力信息，底部="挑战"/"出战"按钮

### 类型F：队伍/配置型（约10%）

用于展示和配置队伍信息。
**典型UI**：UI_TeamInfo、UI_TeamFriend、UI_SingleMarketOut
**特征**：卡片=队员/店铺卡片，底部=操作按钮+属性信息

### 类型G：其他功能型（约10%）

杂项功能界面。
**典型UI**：UI_SettingPush（推送设置开关）、UI_TechnoCourse（课程卡片）、UI_CombineRemould（改造部门）、UI_RelayCommonEquip（装备槽位）

## 常见混淆类型及区分

### vs 上标题-中竖列表-下信息（最重要的区分！）

| 特征 | 中横列表 | 中竖列表 |
|------|---------|---------|
| 列表方向 | **水平**排列（左右滚动） | **垂直**排列（上下滚动） |
| 条目形状 | 竖向卡片 | 横向长条 |
| 可见条目 | 通常露出一排中2-4个 | 列出多行 |
| 滚动方向 | ← → | ↑ ↓ |

**一句话区分**：看中间条目的排列方向——横着排还是竖着排？

### vs 展示-横条-条内信息

| 特征 | 上标题-中横列表-下信息 | 展示-横条-条内信息 |
|------|----------------------|-------------------|
| 形态 | 全屏/大面板式功能界面 | 居中弹窗/浮窗 |
| 背景 | 有背景场景（如店铺街景） | 半透明黑色遮罩 |
| 标题风格 | 功能标题（"选择红颜""出战店铺"） | 结果状态标题（"恭喜！""失败！"） |

### vs 四边木栏（蒸笼标签）+横列表

| 特征 | 上标题-中横列表-下信息 | 四边木栏+横列表 |
|------|----------------------|----------------|
| 边框 | 无木栏边框 | 有四边木栏边框 |
| 挂牌 | 无 | 左侧有挂牌 |
| 标题 | 顶部居中文字标题 | 挂牌替代标题 |

## 主要特征

- 纵向三段式：上标题区 + 中横列表区 + 下信息栏区
- 中部条目水平排列，左右滚动浏览
- 条目通常是竖向卡片（角色/商品/材料卡）
- 底部提供上下文操作（出战/购买/确认）
- 标题通常居中，非贯穿全宽的硬质标题条

## 横竖屏适配要点

- 横屏时横列表可展示更多条目，减少滚动需求
- 标题栏在横屏时更宽，可容纳更多辅助信息
- 底部按钮栏需保持易触达的位置
