# 横列表

## 定义

「横列表」是一体框系列中**横向排列、左右滚动的条目列表**——条目水平展开，每个条目为竖向卡片。与「四边木栏+横列表」的区别：**无木栏边框**。共13个UI。

**与「上标题-中横列表-下信息」的区别**：横列表不一定有上标题+下信息的三段式结构，且属于一体框系列。

## 布局结构

```
┌──────────────────────────────┐
│ ┌────┐ ┌────┐ ┌────┐        │
│ │卡片│ │卡片│ │卡片│ → → → │  ← 横向排列，左右滚动
│ └────┘ └────┘ └────┘        │
└──────────────────────────────┘
```

## 三步判断法（5样本+2验证，盲测2/2通过）

### 第1步：条目是不是横向排列？

- 条目水平排列，左右滚动 → 继续
- 竖向排列 → 不是横列表

### 第2步：有没有木栏边框？

- 无四边木栏 → **横列表** ✓
- 有四边木栏 → 四边木栏（蒸笼标签）+横列表

### 第3步：排除三段式结构

- 有明确的上标题+下信息 → 上标题-中横列表-下信息
- 无三段式 → **横列表** ✓

## 常见混淆

### vs 上标题-中横列表-下信息

| 特征 | 横列表 | 上标题-中横列表-下信息 |
|------|--------|----------------------|
| 上标题 | 不一定 | 必有 |
| 下信息 | 不一定 | 必有 |
| 大类 | 一体框系列 | 上标题系列 |

### vs 四边木栏+横列表

| 特征 | 横列表 | 四边木栏+横列表 |
|------|--------|----------------|
| 边框 | 无木栏 | 四边木栏边框 |

---

## ⚠️ 判断陷阱（补读后补充）

1. **单条目视图**：横列表只有一个条目时，看起来像普通面板（UI_RankEntry、UI_GachaSwitchBeautyPool 空态），容易被误判。需注意右侧留白暗示可滑动空间。
2. **木栏混淆**：部分UI有局部木架装饰（UI_RelayTrainGradeUp），但不是四边全木栏，不能归入四边木栏系列。
3. **伪竖列表**：条目较少时，横列表可能看起来像竖列表（UI_GuildWarSchedule），关键看条目排列方向而非数量。

## 判断口诀

> **条目横向排成行，左右滑动挨个逛。无木无栏无三段，一体横列即此样。**
