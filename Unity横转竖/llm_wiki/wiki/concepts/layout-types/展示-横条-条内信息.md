# 展示-横条-条内信息

## 定义

该布局通过**展示-横条-条内信息**的空间组织方式，实现内容的结果展示和信息呈现。属 展示横条系列 大类，共 63 个 UI，是中等复杂度类型中数量第三多的布局。

**命名含义**：
- **展示**：以信息展示为目的，非操作引导
- **横条**：承载容器为横向条形面板
- **条内信息**：信息内容在横条面板内部垂直排列

## 布局结构

```
┌──────────────────────────┐
│    ✦ 艺术化大标题 ✦     │  ← 装饰性标题（艺术字+丝带/飘带）
│  ┌────────────────────┐  │
│  │                    │  │
│  │   核心信息内容     │  │  ← 横条面板（深色/羊皮纸质感）
│  │   · 状态文字       │  │
│  │   · 奖励图标+数量  │  │
│  │   · 属性/数据      │  │
│  │                    │  │
│  └────────────────────┘  │
│    [按钮]  或  点击关闭  │  ← 底部：按钮 或 关闭提示
└──────────────────────────┘
        半透明黑色遮罩背景
```

## 核心特征（基于63个UI图片分析）

### 1. 必定是居中弹窗/浮窗

100% 的 UI 都是**居中显示**的弹窗形式，背景为半透明黑色遮罩。这是最稳定的识别特征。

### 2. 顶部必有装饰性大标题

绝大多数（约90%）有醒目的艺术化大标题，常见特征：
- 书法体/艺术字体（如"恭喜升级！""战斗结束！""游历结算！"）
- 两侧配丝带/飘带/星光装饰
- 失败界面：灰色/暗色调标题 + "再接再厉""通关失败"等
- 成功界面：金色/亮色调标题 + "恭喜""挑战成功"等

少数（约10%）无标题，仅有彩带装饰（如 UI_BanquetJoinSucc、UI_MuseumGachaResult）。

### 3. 横条面板内容垂直堆叠

信息在横条面板内**从上到下垂直排列**，而非横向平铺：
- 状态描述文字（如"未击败强敌，请再接再厉！"）
- 核心图标/数据（奖励道具图标+数量、角色头像、技能图标等）
- 附加信息（属性加成、进度反馈等）

### 4. 底部一般为"点击关闭"提示或操作按钮

两种底部形态：
- **纯展示型**：底部是"点击空白区域关闭"的提示文字（最常见）
- **操作型**：底部有"确认""再次十连""领取奖励"等功能按钮

## 四种子类型

根据63个UI的语义分析，可分为4种子类型：

### 类型A：奖励结算型（约50%，最多）

**命名特征**：以 `Result`、`Succ`（Success）结尾
**典型UI**：UI_ExpandResult、UI_GuildDungeonBattleSucc、UI_GuestStarSucc、UI_ActSnowWheelResult
**特征**：
- 标题为正向结果（"恭喜！""出海成功！""游历结算！"）
- 中部展示奖励道具（图标+数量）
- 底部为"确认"/"再次"按钮 或 点击关闭提示

### 类型B：失败提示型（约20%）

**命名特征**：以 `Fail` 结尾
**典型UI**：UI_ExpandSeaFailResult、UI_ExpandResultFail、UI_PirateResultFail、UI_RelayTrainRstFail
**特征**：
- 标题为负向结果（"再接再厉！""通关失败！""战斗详情！"）
- 中部展示失败原因 + 解决方案（如"赚速要求：99.99万亿"）
- 常附带跳转引导（"商斗天赋"按钮等）
- 底部为点击关闭提示

### 类型C：信息通知型（约20%）

**命名特征**：无统一的Result/Succ/Fail后缀
**典型UI**：UI_ExpandEventNew、UI_GuestTalentSkill、UI_MountCharactersActive、UI_RoleObtainingTitleTips
**特征**：
- 用于系统通知/信息展示（"发现新事件！""获得称号"）
- 内容为单一信息条目 + 装饰
- 底部一般为点击关闭

### 类型D：进度突破型（约10%）

**命名特征**：含 `Phase`、`Tail`、`Succ` 等
**典型UI**：UI_GuildDungeonBattlePhase、UI_GuildDungeonBattleTail、UI_RelayTrainGradeUpSucc
**特征**：
- 标题为阶段标识（"阶段突破""段位提升"）
- 中部展示阶段/等级变化
- 底部多为"领取奖励"按钮

## 判断流程（三步法）

### 第1步：是否是居中弹窗？

看是否有半透明黑色遮罩背景 + 居中浮窗面板。
- 是 → 继续
- 否 → 不是本类型

### 第2步：是否有装饰性大标题？

顶部是否有醒目的艺术字标题（通常带丝带/飘带装饰）？
- 有大标题 → 很可能是本类型
- 无大标题但有彩带装饰 → 也可能是（少数情况）

### 第3步：内部信息是否在横条面板内垂直排列？

信息内容是否以"文字 → 图标/数据 → 附加信息"的方式在横条内堆叠？
- 是 → **展示-横条-条内信息** ✓
- 否（如：有格子网格、有左右分栏、有多个操作按钮）→ 考虑其他类型

## 常见混淆类型及区分

### vs 确认弹框

| 特征 | 展示-横条-条内信息 | 确认弹框 |
|------|-------------------|----------|
| 目的 | 信息展示/结果通知 | 用户确认选择 |
| 底部按钮 | 单按钮（确认/关闭）或无按钮 | 双按钮（确认+取消） |
| 标题风格 | 结果状态类（"恭喜！""失败！"） | 询问类（"是否XX？"） |
| 装饰程度 | 高度装饰化（丝带、飘带、星光） | 较低装饰 |

### vs 活动打脸弹窗

| 特征 | 展示-横条-条内信息 | 活动打脸弹窗 |
|------|-------------------|-------------|
| 目的 | 系统功能结算 | 活动营销推广 |
| 内容 | 奖励/结算/通知 | 活动详情+引导参与 |
| 视觉 | 简洁的横条面板 | 更丰富的视觉特效+角色 |

### vs 使用弹框

| 特征 | 展示-横条-条内信息 | 使用弹框 |
|------|-------------------|----------|
| 交互 | 被动接收信息（点击关闭即可） | 主动配置参数后执行 |
| 内容 | 结果展示 | 使用数量/配置选项 |
| 按钮 | 单按钮或无 | 确认使用+取消 |

### vs 功能开启&展示

| 特征 | 展示-横条-条内信息 | 功能开启&展示 |
|------|-------------------|--------------|
| 触发时机 | 操作完成后 | 首次解锁时 |
| 内容 | 该次操作的结果 | 功能介绍+引导 |
| 装饰 | 丝带飘带为主 | 可能含角色展示 |

## 判断口诀

> **居中弹窗黑底遮，艺术大字顶上挂。横条面板兜内容，点击空白就关啦。**

1. **居中弹窗**：半透明黑色遮罩+居中面板
2. **艺术大字**：顶部醒目的书法体/美术字标题+丝带装饰
3. **横条面板**：长条形深色/羊皮纸面板承载信息
4. **垂直堆叠**：面板内信息从上到下排列
5. **被动关闭**：底部是"点击关闭"提示或单一确认按钮

## 横竖屏适配要点

- 布局为居中弹窗，横竖屏转换时主要调整弹窗宽度适配
- 关键信息应优先保证可见性
- 奖励图标和文字说明需要保持居中
